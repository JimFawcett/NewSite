Term Definition
AssemblyA compiled unit of code in C#, typically an .exe or .dll file.
AttributeMetadata added to code elements using square brackets, used for reflection or behavior control.
ClassA blueprint for creating objects. Can include fields, properties, methods, and events.
ConstructorA special method that initializes a new instance of a class.
DelegateA type-safe function pointer used to reference methods or implement callbacks/events.
EnumA distinct type representing a set of named constants.
EventA mechanism used to provide notifications, typically implemented with delegates.
ExceptionAn error-handling object used with try, catch, and throw.
FieldA variable defined inside a class or struct.
Garbage CollectionAutomatic memory management that reclaims unused objects.
IndexerAllows objects to be indexed like arrays using the this[] syntax.
InterfaceA contract that defines members a class must implement.
Lambda ExpressionAn anonymous function written using => syntax.
LINQLanguage Integrated Query â€“ a set of query capabilities built into C#.
MethodA function defined within a class or struct.
NamespaceA container for organizing related classes and types.
Nullable TypeA value type that can also hold null, written as int?, bool?, etc.
ObjectAn instance of a class. In C#, all types derive from System.Object.
OverloadingDefining multiple methods with the same name but different parameters.
OverrideReplacing a base class method implementation in a derived class.
PropertyA member that provides controlled access to a field using get and set.
RecordA concise reference type for immutable data, introduced in C# 9.0.
ReflectionThe ability to inspect metadata and interact with objects at runtime.
Sealed ClassA class that cannot be inherited.
StaticIndicates a member or class belongs to the type itself rather than an instance.
StructA value type similar to a class but stored on the stack and without inheritance.
ThreadAn execution path within a program. C# supports multithreading via the System.Threading namespace.
Try-Catch-FinallyControl structure for handling exceptions and cleanup.
UsingUsed for namespace import or to ensure deterministic disposal of resources via IDisposable.
VirtualMarks a method as overrideable in a derived class.
