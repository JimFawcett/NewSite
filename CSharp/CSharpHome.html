1. What is C#?

C# is a general-purpose, type-safe, object-oriented language that compiles to Common Intermediate Language (CIL) bytecode and runs on the .NET Common Language Runtime (CLR). The CLR provides automatic memory management through garbage collection, just-in-time compilation to native code, and a rich reflection API. C# targets Windows, Linux, and macOS from a single codebase and is the primary language for ASP.NET web services, WPF and MAUI desktop/mobile applications, Unity games, and Azure cloud functions. Three defining characteristics:
  • Multi-paradigm - object-oriented classes and interfaces are the foundation, but functional patterns (LINQ, lambdas, records, immutable types) and component-oriented features (attributes, reflection, source generators) are first-class citizens.
  • Managed execution - the CLR handles memory allocation, garbage collection, exception propagation, and type safety at runtime, eliminating entire classes of bugs present in unmanaged languages.
  • Rapid evolution - annual .NET releases since 2020 have added records, pattern matching, primary constructors, collection expressions, generic math, and more, without breaking existing code.
C# was designed by Anders Hejlsberg at Microsoft and first appeared in 2000. The language is standardised (ECMA-334 / ISO/IEC 23270) and its compiler, runtime, and libraries are fully open-source on GitHub under the .NET Foundation.

3. Key Language Concepts

The track covers the areas below, roughly from foundational to advanced.

Types and the .NET Type System

Everything in C# is rooted in System.Object. Understanding value types, reference types, and how the CLR stores and copies them is the first step to writing correct, efficient C#.
  • Value types (struct, primitives, enums) - stored inline; assignment copies all data
  • Reference types (class, arrays, delegates) - heap-allocated; assignment copies the reference
  • Nullable value types (int?) and nullable reference types (C# 8+) - explicit null safety
  • Boxing and unboxing - the cost of treating a value type as object
  • Records (C# 9) - immutable, value-equality reference types; with expressions for non-destructive mutation
Pages: Data, Models, Bites: Data.

Classes and Object-Oriented Programming

C# classes combine encapsulation, inheritance, and polymorphism with modern additions such as properties, init-only setters, and primary constructors.
  • Access modifiers - public, internal, protected, private
  • Properties - auto-properties, computed properties, init setters, required members
  • Single inheritance; multiple interface implementation
  • Virtual dispatch - virtual, override, sealed, abstract
  • Primary constructors (C# 12) - parameters in scope throughout the class body
  • Operator overloading and implicit/explicit conversions
Pages: Classes, Class Relationships.

Generics

C# generics produce a single type-safe definition that the CLR specialises for each type argument at runtime - no code duplication and no boxing for value types.
  • Generic classes and methods with type parameters
  • Constraints - where T : class, struct, new(), interface, base class
  • Covariance (out T) and contravariance (in T) on interfaces and delegates
  • Generic math (C# 11) - INumber<T> and static abstract interface members
  • BCL generic types: List<T>, Dictionary<K,V>, IEnumerable<T>, Task<T>
Pages: Generics, Collections Library.

LINQ and Functional Patterns

Language-Integrated Query (LINQ) is a composable, lazy pipeline of operations over any IEnumerable<T> or IQueryable<T> source.
  • Method syntax - Where, Select, OrderBy, GroupBy, Join, Aggregate
  • Query syntax - SQL-like from … where … select expressions
  • Delegates - Action<T>, Func<T,TResult>, Predicate<T>
  • Lambdas and closures; expression trees (Expression<Func<T,TResult>>)
  • Iterators - yield return / yield break; lazy sequence generation
  • Pattern matching - switch expressions, property, deconstruct, and list patterns
Pages: Operations, Interesting.

Async/Await and Concurrency

The async/await model transforms a method into a state machine that suspends at I/O boundaries without blocking a thread - the primary pattern for scalable servers and responsive UIs.
  • Task and Task<T> - represent in-flight asynchronous operations
  • Task.WhenAll / Task.WhenAny - concurrent fan-out
  • CancellationToken - cooperative cancellation across async call chains
  • IAsyncEnumerable<T> and await foreach - async streaming (C# 8)
  • System.Threading - Thread, Lock, Mutex, SemaphoreSlim, Interlocked
  • Parallel and System.Threading.Channels - data parallelism and producer/consumer pipelines
Pages: Interesting, New Things.

Base Class Library and NuGet

The BCL ships with every .NET installation and covers most common programming tasks. NuGet extends it with thousands of community and Microsoft packages.
  • System.IO - streams, FileStream, StreamReader/Writer, Path, Directory
  • System.Collections.Generic - List<T>, Dictionary<K,V>, HashSet<T>, Queue<T>
  • System.Text.Json - zero-allocation JSON serialization with source generator support
  • System.Reflection - runtime type inspection, dynamic invocation, attributes, emit
  • System.Net.Http - HttpClient for HTTP/REST communication
  • NuGet - dotnet add package <name>; packages for ORM, logging, testing, DI, and more
Pages: Libraries, Streams Library, Collections Library, Reflection.

4. Getting Started

Recommended first steps for someone new to C#:
  1. Install the .NET SDK (version 8 or later). On Windows you may prefer Visual Studio Community (free), which bundles the SDK, debugger, and full IDE. On any platform, Visual Studio Code with the C# extension is an excellent lightweight alternative.
  2. Create a first project with the dotnet CLI: dotnet new console -n Hello && cd Hello && dotnet run. This confirms the tool chain works before writing any real code.
  3. Read C# Bites: Introduction for an overview of what makes C# distinctive and what the track covers.
  4. Work through Bites: Hello C#, then experiment on dotnetfiddle.net for quick online experiments without a local project.
  5. Follow the C# Bites sequence page by page, or read the C# Story in parallel for deeper narrative context.
