# Structure.md — CsTextFinder

*Language- and toolchain-specific layout document for the C# implementation.*

---

## Language & Toolchain

- **Language:** C# 13
- **Runtime:** .NET 10
- **Build:** dotnet CLI or Visual Studio 2022+
- **Nullable:** enabled (`<Nullable>enable</Nullable>`)
- **Implicit usings:** enabled (`<ImplicitUsings>enable</ImplicitUsings>`)

## Component Dependencies

```
CommandLine   DirNav   Output
     \           |       /
      \          |      /
       \         |     /
           EntryPoint
```

Library projects never reference each other or EntryPoint.
Cross-library communication flows through `Action<string>` delegates
registered by EntryPoint at startup.

## Testing

Each library part can be tested with a simple console test project (add with
`dotnet new console -n Tests --output Tests`, then `dotnet sln add Tests/Tests.csproj`
and add `<ProjectReference>` entries for the parts under test).

For a full test framework:

```bash
dotnet new xunit -n Tests --output Tests
dotnet sln add Tests/Tests.csproj
dotnet add Tests/Tests.csproj reference CommandLine/CommandLine.csproj
dotnet test
```
