1.0 Introduction

Code Track provides a place to build and display example code snippets, projects, and tools for code generation. Experiments and demos written in Rust, C++, C#, and Python reside here. It is a place to:
  • experiment with AI support for code development and maintenance
  • Compare implementations of projects in more than one language, e.g., size, complexity, maintainability
  • Build smallish tools for code management
  • Try new language features
Figure 1. CodeWebifier — source-to-HTML conversion tool

2. What is Code Track?

The Code Track is a collection of experiments, demos, and tools focused on writing, analyzing, and maintaining code. It covers multiple programming languages — Rust, C++, C#, and Python — and explores how AI tools can assist throughout the development process. Three defining activities:
  • AI-assisted development - using chat bots, CLI agents, and the Anthropic API to generate, analyze, document, and improve code.
  • Multi-language comparison - implementing the same projects in Rust, C++, C#, and Python to compare design choices, performance, and maintainability across languages.
  • Code tooling - building utilities for code analysis, syntax-highlighted HTML conversion, metrics, and development workflow automation.
The track uses two concrete projects — TextFinder and PageValidator — as recurring examples to keep discussion grounded in real implementations.

4. Key Concepts

The track covers the areas below, each illustrated with working code examples.

AI-Assisted Development

The CodeBites pages document practical workflows for using AI tools in code development.
  • Chat bots - prompting ChatGPT, Claude, and Gemini for code analysis, generation, and documentation
  • CLI agents - interactive sessions with Claude Code and Gemini CLI on real development tasks
  • API agents - building agents with the Anthropic API that read, analyze, and modify code
  • Agentic workflows - multi-step autonomous operations: analyze, plan, generate, test
  • Spec-driven development - using Constitution.md, Structure.md, and Spec.md with AI to drive implementation
Pages: CodeBites Introduction, Chat Bots, Agent AI, Spec-Driven Development.

Multi-Language Projects

Two concrete projects appear throughout the track, each implemented in multiple languages.
  • TextFinder - find regex matches in a directory tree; command-line arguments specify the root path, pattern, and file extensions. Implemented in Rust, RustOpt, C++23, C#, and Python.
  • PageValidator - validate the structural correctness of HTML files in a directory tree. Implemented in Rust, C++, C#, and Python.
  • Utilities - tf_timer.py, pa_timer.py (performance timing), code_metrics.py (line counts and complexity).

Design Structures

The DesignStructure folder contains Rust implementations of five structural patterns applied to the TextFinder domain.
  • BasicStructure - linear package dependencies with clear data flow
  • FactoredStructure - separating policy from mechanism through abstraction
  • DataFlowStructure - components connected by well-typed channels
  • PlugInDataFlowStructure - loading behavior at runtime via dynamic dispatch
  • TypeEraseDataFlowStructure - hiding concrete types behind trait objects

Code Tooling

The track includes utilities that support the development workflow.
  • CodeWebifier - a .NET application that converts source files to syntax-highlighted HTML, used to produce all the code display pages across the site
  • AI Demos - runnable Python scripts for readme generation, code analysis, and structured agent workflows using the Anthropic API

5. Getting Started

Recommended first steps for someone new to the Code Track:
  1. Read the CodeBites Introduction. It lays out the scope of the AI-focused pages and the progression from chat bots through full agentic workflows.
  2. Work through Chat Bots and Code AI for an introduction to prompting and CLI-based AI sessions.
  3. Read Agent AI and Agentic AI to understand how agents are built and how multi-step workflows are structured.
  4. Browse the DesignStructure folder through the Code Explorer to see the five structural patterns in Rust. Read them alongside the SWDev track’s design pattern pages for full context.
  5. Use the Repositories page to find implementations of TextFinder and PageValidator in each language and compare them directly.
