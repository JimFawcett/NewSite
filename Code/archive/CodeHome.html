1. What is Code Track?

Code Track is structured to satisfy three related goals:
  • Act as an access hub for code and code presentations throughout this site.
  • Document recent projects that may still be evolving.
  • Host discussions of Artificial Intelligence (AI) use for working with application code and organization of its documentations.

1.1 Access Hub

TopMenu -> Rust, C++, C#, and Python language-based tracks contain example and demonstration code fragments to explain how those languages work. TopMenu -> WebDev track contains presentations and code examples for web code and UI widgets based on W3C standards. Many of those widgets are used throughout the code tracks presentations. TopMenu -> SWDev track focuses on Design and patterns. TopMenu -> Basics track provides background information about computer platforms. Any code and presentations that don't fit into these tracks are documented here in Code track. Code Track documents active project work across multiple programming languages ‐ Rust, C++, C#, and Python. It is a growing collection; TextFinder and PageValidator are prominent examples. Code Track has four areas of focus:
  • Documenting recent code development - organizes and provides access to descriptions of recent projects and experiments.
  • Multi-language projects - implementing the same problem in Rust, C++, C#, and Python to compare design choices, performance, and maintainability.
  • Code tooling - utilities for syntax-highlighted HTML conversion, code metrics, and development workflow support.
  • AI-assisted development - using chat bots, CLI agents, and the Anthropic API as instruments in the development process.

3. Key Concepts

The track covers the areas below, each illustrated with working code examples.

Multi-Language Projects

Each project is implemented in Rust, C++, C#, and Python, making the design and implementation trade-offs visible across languages. Current projects:
  • TextFinder - find regex matches in a directory tree; command-line arguments specify the root path, pattern, and file extensions. Implemented in Rust, RustOpt, C++23, C#, and Python.
  • PageValidator - validate the structural correctness of HTML files in a directory tree. Implemented in Rust, C++, C#, and Python.
  • Utilities - tf_timer.py, pa_timer.py (performance timing), code_metrics.py (line counts and complexity).
The collection is a starting point. More projects will follow.

Code Tooling

Utilities that support the development and documentation workflow.
  • CodeWebifier - a .NET application that converts source files to syntax-highlighted HTML, used to produce all the code display pages across the site.
  • Performance and metrics scripts - timing harnesses and line-count tools used across projects.

AI-Assisted Development

The CodeBites pages document practical workflows for using AI tools as part of code development.
  • Chat bots - prompting ChatGPT, Claude, and Gemini for code analysis, generation, and documentation
  • CLI agents - interactive sessions with Claude Code and Gemini CLI on real development tasks
  • API agents - building agents with the Anthropic API that read, analyze, and modify code
  • Agentic workflows - multi-step autonomous operations: analyze, plan, generate, test
  • Spec-driven development - using specification documents with AI to drive implementation
Pages: CodeBites Introduction, Chat Bots, Agent AI, Spec-Driven Development.

4. Getting Started

  1. Use the Repositories page to browse current project implementations. Pick a language you know well and read the TextFinder implementation in that language first.
  2. Then read the same project in a second language and compare design choices directly - ownership models in Rust, value semantics in C++, LINQ in C#, list comprehensions in Python.
  3. Browse the Code Explorer to see the CodeWebifier source and understand how the site’s code display pages are generated.
  4. Read the CodeBites Introduction for an overview of the AI-assisted development pages, then work through Chat Bots and Agent AI in order.
