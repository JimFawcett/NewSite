Rules of thumb (applied here)

  1. Single source of truth across boundaries:

    when the same value must be used in multiple places, define a global token. This demo uses :root { --app-gap: 2rem; } and maps it to the spacer with .themed-gap { --h-space-width: var(--app-gap); }.

  2. Attribute vs text vs CSS var:

    width resolution order is CSS var → attribute → text. The CSS variable lets the application override without touching markup.

  3. Display mode:

    spacer defaults to inline-block so it works naturally in a line, flex, or grid. If you want it to behave like a true block (exactly like a raw div), set display="block".
