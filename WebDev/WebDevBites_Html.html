1.0 - HTML Markup

Hyper Text Markup Language (HTML) is a declarative language specifing what will be placed on a web page and, with some exceptions, where it will be rendered. HTML markup consists of a tree of elements, as shown in Figure 1a. All html elements are wrapped in Tags.
Definition HTML current specification HyperText Markup Language (HTML) is a language for structuring content on a web page. It is based on Elements that have opening and closing tags and bodies with content. 
  <htmltag>              <!-- any of the standard html tags, e.g., div -->
    <custom-childtag>    <!-- a user defined tag with some semantic meaning -->
      "literal text"
    </custom-childtag>
  </htmltag>
  • An element body, everything between opening and closing tags, may consist of text or markup or both.
  • The HTML standard requires that all custom tags are hypenated, as shown above, to avoid clashes with future standard tags.
HTML elements are divided into two groups: in-line and block. These categories determine how rendering flow appears on a web page.
  • inline-elements flow horizontally from page left to right
    span, a, br, strong, sub, sup, del, ...
  • block elements flow vertically from page top to bottom
    html, body, head, header, script, style, main, footer, section, svg, img,
    table, tr, th, td,
    pre, code, div, p, h1, ... h6, ol, ul, li, nav,
    form, iframe, input
    ...
HTML pages are usually structured like this: 
  <!DOCTYPE html>
  <html>
    <head>
      <style>
        /*-- linked and inline styles go here --*/
      </style>
      <script>
        <!-- linked and inline scripts go here -->
      </script>
    </head>
    <body>
      <!-- content markup goes here -->
      <script>
        <!-- inline scripts that run at load time go here -->
      </script>
    </body>
  </html>
HTML events are generated by browser and user actions, e.g., page load or button click. They can be used as HTML attributes like this: 
<div onclick="doShow()" ... >text content</div>
where doShow() is some JavaScript function defined in the current page.

2.0 - HTML Models

Rendering of HTML into visible page contents is governed by several models:
Page Model Governs how a web page must be structured
Flow Model Determines how elements are placed visually on a web page
Box Model Defines how elements with display:block are rendered

2.1 - Page Model

Figure 1. Page Structure 
<!DOCTYPE html>
<html id="top" lang="en">
  <head>
    <!-- Examples::PageModel.html -->
    <meta charset="UTF-8">
    <title>Page Model</title>
    <link rel="stylesheet" href="css/reset.css"></link>
    <link rel="stylesheet" href="css/content.css">
    <link rel="stylesheet" href="css/Example.css">
    <link rel="stylesheet" href="css/PageModel.css">
  </head>
  <body>
  <!-- rendered markup starts here ----------------------->
      <h3>PageModel.html</h3>
      <section>
        Contains first section's content.
      </section>
      <section>
        Another section with child elements:
        <div>child 1</div>
        <div>child 2</div>
      </section>
    <a id="bottom"></a>
  <!-- rendered markup ends here ------------------------->
  </body>
</html>
A properly structured static web page is a tree of HTML elements, rooted at <html> called the Document Object Model (DOM). That contains two elements: <head> and <body>.
Figure 1a - HTML DOM
<head> contains:   
  • Declarations and metadata
  • Links to style sheets
  • inline styles
  • Links to JavaScript files
  • inline JavaScript
 <body> contains:
  • html elements
  • script blocks
 The <head> elements alter how body elements are styled and react to events, but are not directly responsible for rendering content. That is, if the body contents were empty, the rendered page would be blank. All the html elements in the <body> are rendered to the page except when they have type display: none. In Figure 1a we show two script elements with dotted ownership since the example of Figure 1 has no script elements. Those elements show that script can be placed in either the head or the body, usually at the end. The styles in Figure 1a. can be placed in linked files or in <style> blocks within the head element. They also can be declared inline as part of the styled element in an style="a style" attribute. Similarly, scripts can be placed in a source file or in a <script> block within the head. Script function invocations are often associated with element clicks with a onclick="some_function()" attribute, on elements in the <body>.

2.2 - Flow Model

Figure 2. Flow Model Markup 
<!DOCTYPE html>
<html id="mkup" lang="en">
<head>
  <!-- Examples::FlowModel.html -->
  <meta charset="UTF-8">
  <title>Flow Model</title>
  <link rel="stylesheet" href="css/reset.css"></link>
  <link rel="stylesheet" href="css/content.css">
  <link rel="stylesheet" href="css/Example.css">
  <link rel="stylesheet" href="css/FlowModel.css">
</head>
<body>
  <!-- rendered markup starts here ----------------------->
  <h3>FlowModel.html</h3>
  <span>A</span>
  <span>B</span>
  <span>C</span>
  <span>D</span><span>E</span><span>F</span>
  <div class="parent">
    Parent div
    <div class="child">Child div</div>
  </div>
  <div class="parent">
    Sibling of Parent div
  </div>
  <div class="parent" style="margin:0.5rem 0rem;">
    Sibling div with margin
  </div>
  <!-- rendered markup ends here ------------------------->
</body>
Figure 2a - Flow Model
The Flow Model governs how html elements are rendered onto a static web page:
  • Elements with display:block, like <div>, are placed vertically, in the order defined by the page's code.
    No space is inserted between block elements unless they have a margin style.
  • Elements with display:inline, like <span>, are placed horizontally until they use up their container's width.
    Without margin styling no space is placed between the elements unless they wrap or are manually placed in a new line.

2.3 - Box Model

Figure 3. Box Model 
<!DOCTYPE html>
    <html id="top" lang="en">
      <head>
        <!-- Examples::BoxModel.html -->
        <meta charset="UTF-8">
        <title>Box Model</title>
        <link rel="stylesheet" href="css/reset.css"></link>
        <link rel="stylesheet" href="css/content.css">
        <link rel="stylesheet" href="css/Example.css">
        <link rel="stylesheet" href="css/BoxModel.css">
        <script src="js/BoxModel.js"></script>
      </head>
      <body onload="load()">
        <h3>BoxModel.html</h3>
        <div id="box-demo">
          <p id="content-demo">
            Content within a paragraph in a div box.
          </p>
        </div>
        <table>
          <tr>
            <td>content width</td>
            <td id="para"></td>
          </tr>
          <tr>
            <td>box width</td>
            <td id="div"></td>
          </tr>
          <tr>
            <td>padding width</td>
            <td id="padding"></td>
          </tr>
          <tr>
            <td>border width</td>
            <td id="border"></td>
          </tr>
        </table>
        <t-b>
          Numerical content in table above was generated by functions in <c-s>BoxModel.js</c-s>.
        </t-b>
        <div style="height:2rem;"></div>
      </body>
    </html>
The Box Model governs how block-structured html elements are rendered onto a static web page:
  1. Elements with display:block, like <div>, may have borders, padding, and content.
    The box-sizing style has two values:
    • content-box (default). Styles width and height apply only to the content. This is the default for historical reasons, but is usually overridden with box-sizing: border-box;
    • border-box. Styles width and height apply to the box's border.
  2. Elements with display:inline, like <span>, do not respond to height and width styles.
    • Vertical sizing is affected by line-height and font-size.
    • Horizontal spacing is modified with padding-left/right and margin-left/right. Vertical margins are ignored.
In Figure 3., the <div id="box-demo"> defines a box with border, padding, and a paragraph of content. The border and padding and background colors are defined in the stylesheet BoxModel.css.
