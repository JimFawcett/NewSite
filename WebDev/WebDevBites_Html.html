1.0 - HTML Markup

Web page content is defined with HTML markup, styled with CSS, and animated with JavaScript. This page will focus on HTML markup. HTML markup consists of a tree of elements. All html elements are wrapped in Tags:
<div> --some content-- </div>
Some are self closing:
<br>
A complete tag listing is given in drop-down, below.

2.0 - HTML Models

Rendering of HTML into visible page contents is governed by several models:
Page Model Governs how a web page must be structured
Flow Model Determines how elements are placed visually on a web page
Box Model Defines how elements with display:block are rendered

2.1 - Page Model

Figure 1a - HTML DOM
Figure 1. Page Structure 
<!DOCTYPE html>
<html id="top" lang="en">
  <head>
    <!-- Examples::PageModel.html -->
    <meta charset="UTF-8">
    <title>Page Model</title>
    <link rel="stylesheet" href="css/reset.css"></link>
    <link rel="stylesheet" href="css/content.css">
    <link rel="stylesheet" href="css/Example.css">
    <link rel="stylesheet" href="css/PageModel.css">
  </head>
  <body>
  <!-- rendered markup starts here ----------------------->
      <h3>PageModel.html</h3>
      <section>
        Contains first section's content.
      </section>
      <section>
        Another section with child elements:
        <div>child 1</div>
        <div>child 2</div>
      </section>
    <a id="bottom"></a>
  <!-- rendered markup ends here ------------------------->
  </body>
</html>
A properly structured static web page is a tree of HTML elements, rooted at <html> called the Document Object Model (DOM). That contains two elements: <head> and <body>. <head> contains:
  • Declarations and metadata
  • Links to style sheets
  • inline styles
  • Links to JavaScript files
  • inline JavaScript
 <body> contains:
  • html elements
  • script blocks
 The <head> elements alter how body elements are styled and react to events, but are not directly responsible for rendering content. That is, if the body contents were empty, the rendered page would be blank. All the html elements in the <body> are rendered to the page except when they have type display: none. In Figure 1a we show two script elements with dotted ownership since the example of Figure 1 has no script elements. Those elements show that script can be placed in either the head or the body, usually at the end. The styles in Figure 1a. can be placed in linked files or in <style> block within the head element. They also can be declared inline as part of the styled element in an style="a style" attribute. Similarly, scripts can be placed in a source file or in a <script> block within the head. Script function invocations are often associated with element clicks with a onclick="some_function()" attribute, on elements in the <body>.

2.2 - Flow Model

Figure 2. Page Flow 
<!DOCTYPE html>
  <html id="mkup" lang="en">
  <head>
    <!-- Examples::FlowModel.html -->
    <meta charset="UTF-8">
    <title>Flow Model</title>
    <link rel="stylesheet" href="css/reset.css"></link>
    <link rel="stylesheet" href="css/content.css">
    <link rel="stylesheet" href="css/Example.css">
    <link rel="stylesheet" href="css/FlowModel.css">
  </head>
  <body>
    <!-- wrappers seperate rendered markup from comments --->
    <body-wrapper>
      <code-wrapper>
    <!-- rendered markup starts here ----------------------->
        <h3>FlowModel.html</h3>
        <span>A</span>
        <span>B</span>
        <span>C</span>
        <span>D</span><span>E</span><span>F</span>
        <div class="parent">
          Parent div
          <div class="child">Child div</div>
        </div>
        <div class="parent">
          Sibling of Parent div
        </div>
        <div class="parent" style="margin:0.5rem 0rem;">
          Sibling div with margin
        </div>
    <!-- rendered markup ends here ------------------------->
      </code-wrapper>
      <comment-wrapper>
        Comments:
        <t-b>
          Elements A ... F are <code>display:inline</code>
          so layout flows left to right.
        </t-b>
        <t-b>
          Parent and Siblings are <code>display:block</code>
          so layout flows top to bottom.
        </t-b>
        <t-b>
          Child is declared inside Parent, so starts there
          in flow order, but is not bound to display only
          inside Parent.
        </t-b>
        <t-b>
          Divs flow top-to-bottom with no spaces unless they
          have margin styling.
        </t-b>
      </comment-wrapper>
    </body-wrapper>
  </body>
Figure 2a - Flow Model
The Flow Model governs how html elements are rendered onto a static web page:
  • Elements with display:block, like <div>, are placed vertically, in the order defined by the page's code.
    No space is inserted between block elements unless they have a margin style.
  • Elements with display:inline, like <span>, are placed horizontally until they use up their container's width.
    Without margin styling no space is placed between the elements unless they wrap or are manually placed in a new line.

2.3 - Box Model

Figure 3a - Box Model Demo
Figure 3. Box Model 
<!DOCTYPE html>
    <html id="top" lang="en">
      <head>
        <!-- Examples::BoxModel.html -->
        <meta charset="UTF-8">
        <title>Box Model</title>
        <link rel="stylesheet" href="css/reset.css"></link>
        <link rel="stylesheet" href="css/content.css">
        <link rel="stylesheet" href="css/Example.css">
        <link rel="stylesheet" href="css/BoxModel.css">
        <script src="js/BoxModel.js"></script>
      </head>
      <body onload="load()">
        <h3>BoxModel.html</h3>
        <div id="box-demo">
          <p id="content-demo">
            Content within a paragraph in a div box.
          </p>
        </div>
        <table>
          <tr>
            <td>content width</td>
            <td id="para"></td>
          </tr>
          <tr>
            <td>box width</td>
            <td id="div"></td>
          </tr>
          <tr>
            <td>padding width</td>
            <td id="padding"></td>
          </tr>
          <tr>
            <td>border width</td>
            <td id="border"></td>
          </tr>
        </table>
        <t-b>
          Numerical content in table above was generated by functions in <c-s>BoxModel.js</c-s>.
        </t-b>
        <div style="height:2rem;"></div>
      </body>
    </html>
The Box Model governs how block-structured html elements are rendered onto a static web page:
  1. Elements with display:block, like <div>, may have borders, padding, and content.
    The box-sizing style has two values:
    • content-box (default). Styles width and height apply only to the content. This is the default for historical reasons, but is usually overridden with box-sizing: border-box;
    • border-box. Styles width and height apply to the box's border.
  2. Elements with display:inline, like <span>, do not respond to height and width styles.
    • Vertical sizing is affected by line-height and font-size.
    • Horizontal spacing is modified with padding-left/right and margin-left/right. Vertical margins are ignored.
In Figure 3., the <div id="box-demo"> defines a box with border, padding, and a paragraph of content. The border and padding and background colors are defined in the stylesheet BoxModel.css. A complete example is given here: Box Model in a new tab.
