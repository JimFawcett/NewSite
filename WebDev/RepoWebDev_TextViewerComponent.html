WebDev Text Viewer Component

Expandable view that may contain <pre> blocks

About Repositories

1.0 Overview

TextViewerComponent is a zero-dependency vanilla JavaScript W3C Web Component that displays a titled, framed block of text or pre-formatted content with click-to-resize interaction. It is implemented as a Custom Element with Shadow DOM encapsulation so it works as a native HTML element in any modern browser with no framework required.
The component accepts arbitrary HTML in its content slot, making it suitable for plain text, pre-formatted output, or syntax-highlighted code when combined with Prism.js. Clicking the content area widens it; clicking the title bar narrows it. Width is capped at 80% of the viewport width.

2.0 Demo

Open TextViewerComponent.html in a browser to see the component floating beside body text with interactive width control.

3.0 Installation

Copy js/TextViewerComponent.js into your project and load it with a script tag. No build step or package manager required. 
<script src="path/to/TextViewerComponent.js" defer></script>
For Prism syntax highlighting in the content slot, also load Prism before the component script: 
<link rel="stylesheet" href="path/to/prism.css">
<script src="path/to/prism.js"></script>

4.0 Usage

<text-viewer
  width="25rem"
  title-bg-color="#bbb"
  bg-color="#f9f9f9"
  style="float: right;">

  <div slot="title">Fig 1. &mdash; Build Output</div>

  <div slot="content">
    <pre><code>Compiling hello v0.1.0
Finished dev [unoptimized] target(s) in 0.42s
Running `target/debug/hello`
Hello, world!</code></pre>
  </div>
</text-viewer>
With Prism syntax highlighting: 
<text-viewer width="30rem" title-bg-color="#ccc">
  <div slot="title">Fig 2. &mdash; Rust example</div>
  <div slot="content">
    <pre><code class="language-rust">fn main() {
    println!("Hello, world!");
}</code></pre>
  </div>
</text-viewer>

5.0 API

Attributes are read once when the element is constructed. Set them in HTML before the element is registered, or via setAttribute before the element is connected to the DOM.
Attributes
AttributeDefaultDescription
widthmax-contentWidth of the component frame. Accepts any CSS length: px, rem, %, etc.
heightmax-contentHeight of the component frame. Use a fixed value (e.g. 20rem) to create a scrollable block.
bg-colorwhiteBackground color of the component frame.
title-bg-colortransparentBackground color of the title bar.
Slots
SlotDefault contentDescription
title"Text Viewer"Title bar content. Accepts any inline HTML.
content(empty)The text or code to display. Accepts any HTML; white-space: pre is applied so raw text and <pre> blocks render without collapsing whitespace.
Place the title in a <div slot="title"> and the content in a <div slot="content">: 
<text-viewer>
  <div slot="title">Caption here</div>
  <div slot="content">Text here</div>
</text-viewer>
CSS Custom Properties
The component reads two CSS custom properties from the host page for theme integration:
PropertyUsed forFallback
--lightOuter wrapper backgroundwhite
--darkBorder color and title text#333
:root {
  --light: #f5f0e8;
  --dark:  #2c2c2c;
}
Shadow DOM Parts
Two named parts are exposed for external styling: 
text-viewer::part(title) {
  font-size: 0.85rem;
  letter-spacing: 0.04em;
}

text-viewer::part(content) {
  font-size: 0.9rem;
  line-height: 1.5;
}
Interaction
ActionEffect
Click the content areaWidens the component by 20% (scale factor ×1.2).
Click the title barNarrows the component by ~17% (scale factor ÷1.2).
Width is proportionally scaled from the current rendered width on each click. The minimum width is 10 px and the maximum is 80% of the current viewport width. The component animates width changes with a 0.2 s ease transition.

6.0 Browser Support

TextViewerComponent uses three W3C Web Component specifications:
SpecificationPurpose
Custom Elements v1 Defines <text-viewer> as a native HTML element
Shadow DOM v1 Encapsulates internal styles and structure
HTML Slots Exposes the title and content insertion points
Supported in all modern browsers (Chrome, Edge, Firefox, Safari).
