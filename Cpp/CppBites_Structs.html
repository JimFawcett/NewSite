C++ Bites: Structs

aggregates, designated initializers, and POD types

About C++ Track

1.0 Synopsis

A C++ struct is a class whose members are public by default. Structs work well as plain data holders (aggregates) that benefit from brace initialization, C++20 designated initializers, and structured bindings. The conventional distinction: use struct for data, class for types that enforce invariants through private members and constructors.

2.0 Struct Declaration

A struct can contain data members, member functions, constructors, and static members - everything a class can. The only language difference between struct and class is default access: public for struct, private for class. 
struct Color {
  uint8_t r = 0;
  uint8_t g = 0;
  uint8_t b = 0;
  uint8_t a = 255;

  // member function
  bool isOpaque() const { return a == 255; }
};

Color red = {255, 0, 0};       // aggregate init
Color transparent = {0,0,0,0};
std::cout << red.isOpaque() << '\n';  // 1

3.0 Aggregate Initialization

An aggregate is a class type with no user-provided constructors, no private or protected non-static data members, no virtual functions, and no virtual, private, or protected base classes (the definition was relaxed in C++17 to allow public base classes). Aggregates can be initialized with a brace-enclosed list in declaration order - no constructor call required. Members omitted from the initializer list are value-initialized (zeroed for scalars, default-constructed for class types). Default member initializers (= value in the member declaration) are used for any member not explicitly provided. 
struct Point3D {
  double x = 0.0;
  double y = 0.0;
  double z = 0.0;
};

// aggregate initialization
Point3D a = {1.0, 2.0, 3.0};

// C++20 designated initializers
Point3D b = {.x = 1.0, .z = 3.0};  // y defaults to 0.0

// structured binding
auto [x, y, z] = a;

4.0 Designated Initializers (C++20)

Designated initializers allow naming each field being initialized. The advantages over positional initialization are clarity (the field name is visible at the call site) and tolerance of field additions (new fields default-initialize unless explicitly named). Fields must be specified in declaration order; skipping fields is allowed. 
struct Config {
  int    timeout  = 30;
  bool   verbose  = false;
  int    retries  = 3;
  double interval = 1.0;
};

// only specify what differs from defaults
Config c = {
  .timeout  = 60,
  .verbose  = true,
  // retries and interval keep their defaults
};

// positional would be error-prone with 4+ fields
// Config c2 = {60, true};  // compiles but fragile

5.0 POD and Trivially Copyable

A POD (Plain Old Data) type satisfies two requirements: it is trivially copyable (copy and move operations are bitwise copies; destructor does nothing non-trivial) and it has standard layout (no virtual functions, no mixed access specifiers in the same class, members in predictable order). POD types can be safely memcpy'd, serialized byte-by-byte, and passed to C APIs. Type traits in <type_traits> let you check these properties at compile time: 
#include <type_traits>

struct Vec2 { float x; float y; };

static_assert(std::is_trivially_copyable_v<Vec2>);
static_assert(std::is_standard_layout_v<Vec2>);

// std::string is NOT trivially copyable
static_assert(!std::is_trivially_copyable_v<std::string>);

// safe to memcpy a POD
Vec2 a = {1.0f, 2.0f};
Vec2 b;
std::memcpy(&b, &a, sizeof(Vec2));

6.0 Nested Structs

Structs can nest freely. Nested types are accessed via the outer type's scope. Aggregate initialization can initialize nested aggregates with inner brace groups. 
struct Point {
  double x = 0.0;
  double y = 0.0;
};

struct Segment {
  Point start;
  Point end;

  double length() const {
    double dx = end.x - start.x;
    double dy = end.y - start.y;
    return std::sqrt(dx*dx + dy*dy);
  }
};

// nested aggregate initialization
Segment s = {{0.0, 0.0}, {3.0, 4.0}};
std::cout << s.length() << '\n';  // 5.0

// C++20 designated
Segment t = {.start = {.x = 1.0, .y = 1.0}, .end = {.x = 4.0, .y = 5.0}};
