C++ Bites: Tips and Tricks

structured bindings, fold expressions, if constexpr, and modern utilities

About C++ Track

1.0 Synopsis

Modern C++ (C++17 and C++20) adds concise, expressive features that reduce boilerplate and improve clarity. This page collects practical tips organized by feature area. All examples compile with -std=c++20; features introduced in C++17 are noted inline.

2.0 Structured Bindings (C++17)

auto [a, b] = expr unpacks a pair, tuple, array, or aggregate into named variables. The bindings are references to the underlying members when declared with auto&. Structured bindings are most useful in range-for over maps and when returning multiple values from a function. 
std::map<std::string, int> scores{{"Alice", 95}, {"Bob", 87}};

for (auto& [name, score] : scores)     // structured binding
  std::cout << name << ": " << score << '\n';

if (auto it = scores.find("Alice"); it != scores.end())
  std::cout << it->second << '\n';   // 95

// unpack a pair returned from insert
auto [iter, inserted] = scores.emplace("Carol", 91);

3.0 if and switch with Initializer (C++17)

if (init; condition) scopes the initialized variable to the if/else block, preventing accidental use after the condition. It also eliminates the need for a separate declaration line before the if. The same syntax works in switch. 
// variable 'it' is scoped to the if/else block
if (auto it = scores.find("Bob"); it != scores.end()) {
  it->second += 5;    // in-scope here
} // 'it' goes out of scope here

// switch with initializer
switch (int status = getStatus(); status) {
  case 0:  handleOk();    break;
  case -1: handleError(); break;
}

4.0 if constexpr (C++17)

Inside a template, if constexpr evaluates the condition at compile time and discards the non-taken branch entirely - it is not even required to be syntactically valid for the instantiated type. This replaces most uses of SFINAE for selecting type-specific code paths. 
template<typename T>
void describe(T x) {
  if constexpr (std::is_integral_v<T>)
    std::cout << x << " is an integer\n";
  else if constexpr (std::is_floating_point_v<T>)
    std::cout << x << " is floating point\n";
}

describe(42);      // "42 is an integer"
describe(3.14);    // "3.14 is floating point"

5.0 Fold Expressions (C++17)

Fold expressions apply a binary operator across a parameter pack. (args + ...) is a unary right fold; (... + args) is a unary left fold. They replace recursive template implementations for variadic operations such as sum, print, and logical AND/OR. 
template<typename... Ts>
auto sum(Ts... args) { return (args + ...); }

std::cout << sum(1, 2, 3, 4) << '\n';  // 10

// print all args separated by spaces
template<typename... Ts>
void printAll(Ts... args) {
  ((std::cout << args << ' '), ...);
  std::cout << '\n';
}

// check all conditions
template<typename... Bools>
bool allTrue(Bools... bs) { return (bs && ...); }

6.0 [[nodiscard]] and [[maybe_unused]]

[[nodiscard]] causes the compiler to warn when the annotated function's return value is discarded without being used. Apply it to functions whose return value carries an error code or a resource that must be released. [[maybe_unused]] suppresses unused-variable warnings for variables that are intentionally present (e.g., debug-only parameters). 
[[nodiscard]] int openFile(const char* path);

openFile("data.txt");    // warning: result discarded

void compute([[maybe_unused]] int debugLevel) {
  // debugLevel only used in debug builds
}

7.0 std::exchange

std::exchange(obj, newVal) assigns newVal to obj and returns the previous value of obj. It is particularly useful in move constructors and move assignment operators to transfer a value while simultaneously setting the source to a safe default. 
#include <utility>

struct Handle {
  int fd = -1;

  Handle(Handle&& other) noexcept
    : fd(std::exchange(other.fd, -1)) {}  // take fd, leave other invalid

  ~Handle() { if (fd != -1) close(fd); }
};

8.0 CTAD - Class Template Argument Deduction (C++17)

CTAD lets the compiler deduce template arguments from constructor arguments, removing the need to spell out the type explicitly. It works for most standard containers, std::pair, std::tuple, std::lock_guard, and user-defined types with deduction guides. 
// Before C++17 - explicit template arguments required
std::vector<int> v1{1, 2, 3};
std::pair<std::string, int> p1{"Alice", 42};

// C++17 CTAD - deduced from arguments
std::vector v2{1, 2, 3};           // std::vector<int>
std::pair   p2{"Alice", 42};       // std::pair<const char*, int>
std::array  a{1.0, 2.0, 3.0};     // std::array<double, 3>

std::mutex mtx;
std::lock_guard lk(mtx);          // std::lock_guard<std::mutex>
