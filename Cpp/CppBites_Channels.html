C++ Bites: Channels

message passing with thread-safe queues

About C++ Track

1.0 Synopsis

C++ has no built-in channel type comparable to Go channels or Rust's std::sync::mpsc. The standard building blocks — std::queue, std::mutex, and std::condition_variable — combine straightforwardly into a reusable thread-safe channel template. This pattern is the idiomatic C++ approach to producer-consumer message passing.

2.0 Thread-safe Queue

The Channel<T> template below wraps a std::queue<T> with a mutex and condition variable. send pushes under the lock and notifies one waiting receiver. receive waits until the queue is non-empty, then moves the front element out. Moving avoids an extra copy for non-trivial types. 
#include <queue>
#include <mutex>
#include <condition_variable>

template<typename T>
class Channel {
public:
  void send(T value) {
    std::unique_lock<std::mutex> lock(mtx_);
    queue_.push(std::move(value));
    cv_.notify_one();
  }
  T receive() {
    std::unique_lock<std::mutex> lock(mtx_);
    cv_.wait(lock, [this]{ return !queue_.empty(); });
    T value = std::move(queue_.front());
    queue_.pop();
    return value;
  }
private:
  std::queue<T>           queue_;
  std::mutex               mtx_;
  std::condition_variable  cv_;
};

3.0 Producer-Consumer Pattern

A common pattern sends a sentinel value to signal that the producer is finished. The consumer checks each received item and exits its loop when the sentinel arrives. std::optional<T> works well as a sentinel carrier — an empty optional signals end-of-stream without requiring a magic domain value. 
#include <optional>
#include <thread>
#include <iostream>

int main() {
  Channel<std::optional<int>> ch;

  std::thread producer([&ch]() {
    for (int i = 0; i < 5; ++i)
      ch.send(i);
    ch.send(std::nullopt);  // sentinel: end of stream
  });

  std::thread consumer([&ch]() {
    while (true) {
      auto item = ch.receive();
      if (!item) break;           // sentinel received
      std::cout << *item << '\n';
    }
  });

  producer.join();
  consumer.join();
}

4.0 Multiple Consumers

Multiple threads can all call receive() on the same channel safely. The mutex serializes access so only one thread dequeues a given item. The condition variable wakes one waiter per notify_one() call, distributing work naturally across consumer threads. Use notify_all() if all waiters need to re-evaluate their predicate (for example, when sending a shutdown signal to every consumer). 
// Each consumer thread runs this loop independently
auto worker = [&ch]() {
  while (true) {
    auto item = ch.receive();
    if (!item) {
      ch.send(std::nullopt);  // re-post sentinel for other consumers
      break;
    }
    process(*item);
  }
};

std::thread t1(worker), t2(worker), t3(worker);
ch.send(std::nullopt);  // one sentinel starts the shutdown chain
t1.join(); t2.join(); t3.join();
