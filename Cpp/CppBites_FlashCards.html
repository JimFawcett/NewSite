C++ Bites: Flash Cards

key terms and concepts for quick review

About C++ Track

1.0 Synopsis

This is a quick-reference set of flash cards covering essential C++ terms and concepts. Click any card to reveal its definition. Cards are ordered from foundational to advanced. Use them to test recall before an interview or to sharpen understanding of language mechanics.

2.0 Cards

RAII   Resource Acquisition Is Initialization. Resources (memory, file handles, sockets, locks) are acquired in a constructor and released in the destructor. Because the compiler guarantees destructor calls when an object leaves scope - including during stack unwinding from an exception - resources are never leaked regardless of the control flow path taken.
Rule of Five   If a class needs a user-defined destructor, copy constructor, or copy assignment operator, it almost certainly needs all five special member functions: destructor, copy constructor, copy assignment, move constructor, and move assignment. Declaring any one of them suppresses compiler generation of the move operations, so all five must be provided explicitly.
Rule of Zero   If a class manages no raw resources directly - delegating ownership to member objects such as std::string, std::vector, or std::unique_ptr - declare none of the five special member functions and let the compiler generate correct implementations for all of them. This is the preferred modern approach.
ODR - One Definition Rule   Every non-inline function, variable, or class used in a program must have exactly one definition across all translation units. Classes and inline functions may be defined in multiple translation units provided all definitions are identical. Violations produce undefined behavior (often a linker error for functions, silent misbehavior for types).
ADL - Argument-Dependent Lookup   Also called Koenig lookup. When a function is called without explicit namespace qualification, the compiler searches not only the enclosing scopes but also the namespaces associated with the types of the function arguments. This makes operator<< and swap work naturally without requiring std:: prefixes at call sites.
SFINAE - Substitution Failure Is Not An Error   When the compiler attempts to substitute template arguments and the substitution produces an invalid type or expression, that candidate is silently removed from the overload set rather than causing a compilation error. This is the mechanism behind std::enable_if and many pre-C++20 template constraints. C++20 Concepts replace most SFINAE idioms with cleaner syntax.
decltype   A type specifier that queries the declared type of an expression without evaluating it. decltype(expr) preserves reference qualifiers: if expr is an lvalue it yields an lvalue reference type; if it is an xvalue it yields an rvalue reference type. Commonly used in trailing return types and generic code: auto f() -> decltype(x + y).
auto   Instructs the compiler to deduce the type of a variable from its initializer. Since C++11 for variables and C++14 for return types. auto strips top-level cv-qualifiers and references; write auto& or const auto& to preserve them. auto in a lambda parameter list (C++14) creates a generic lambda.
constexpr   Applied to a variable or function, constexpr indicates that the entity can be evaluated at compile time when given constant-expression arguments. A constexpr function may also be called at runtime. Since C++14, constexpr functions can contain loops and local variables. Replaces unsafe #define constants with type-safe alternatives.
consteval (C++20)   Declares an immediate function that must be evaluated at compile time; calling it at runtime is an error. Stronger than constexpr, which permits runtime calls. Use consteval when the result must be a compile-time constant, for example when generating lookup tables or validating arguments at compile time.
noexcept   A specifier that declares a function will not throw exceptions. If a noexcept function does throw, std::terminate is called immediately. Marking move constructors and move assignment operators noexcept allows STL containers to use them during reallocation instead of falling back to copies. noexcept(expr) is also an operator that evaluates to true if expr is declared noexcept.
[[nodiscard]] (C++17)   An attribute that causes a compiler warning when the return value of a function (or a value of the attributed type) is discarded. Apply to functions whose return value carries an error code, an owned resource, or a result that must be checked - for example [[nodiscard]] bool try_lock(). C++20 allows adding a string reason: [[nodiscard("check the error code")]].
std::move   A cast, not a move. std::move(x) is equivalent to static_cast<std::remove_reference_t<decltype(x)>&&>(x) - it converts an lvalue into an rvalue reference, signaling to the compiler that the caller is done with the object and its resources may be transferred. The actual transfer happens in the move constructor or move assignment operator that is subsequently called.
std::forward   Preserves the value category (lvalue or rvalue) of a forwarding-reference argument. Used inside function templates that accept T&& parameters to pass arguments to another function without losing their original lvalue or rvalue nature. Without std::forward, a named parameter is always an lvalue even if the caller passed an rvalue, and move semantics would be lost.
Perfect Forwarding   The pattern of accepting arguments as forwarding references (T&& in a template) and passing them to another callable with std::forward<T>(arg). This preserves both the type and value category, allowing a wrapper function to be indistinguishable from a direct call at the point where the ultimate target is invoked. Used extensively in factory functions, emplace, and wrapper utilities.
Copy Elision / RVO / NRVO   Copy elision allows the compiler to construct a returned or thrown object directly in the destination storage, eliminating the copy or move entirely. RVO (Return Value Optimization) applies when a function returns a temporary (prvalue); since C++17 this is mandatory. NRVO (Named RVO) applies when a named local variable is returned; it is permitted but not required. Both can make code that looks like it copies actually perform zero copies.
Aggregate   A class or array with no user-provided constructors, no private or protected non-static data members, no base classes (until C++17 extended the rules), and no virtual functions. Aggregates support aggregate initialization with braced lists: Point p = { 1, 2 };. The rules were progressively relaxed in C++11 through C++20.
POD - Plain Old Data   A type that is both trivial and standard-layout. Trivial means it has trivial special member functions. Standard-layout means its members are laid out compatibly with C structs. POD types can be safely memcpy-ed, serialized as raw bytes, and passed across C/C++ ABI boundaries. The formal POD concept was deprecated in C++20 in favor of the more precise trivial and standard-layout concepts.
Trivially Copyable   A type whose copy/move constructors and copy/move assignment operators are trivial (compiler- generated or explicitly defaulted with no side effects) and whose destructor is trivial. Trivially copyable types may be copied with std::memcpy. The standard library uses this property to optimize algorithms - for example, std::copy reduces to memcpy for trivially copyable element types.
ABI - Application Binary Interface   The low-level contract governing how compiled code interacts at the binary level: calling conventions, name mangling, struct layout, exception handling tables, and vtable structure. C++ has no standard ABI; each platform/compiler combination defines its own. Breaking ABI compatibility (changing a virtual function or adding a non-static data member to a class used across a shared-library boundary) requires recompiling all consumers. The Itanium C++ ABI is used by GCC and Clang on Linux; MSVC uses its own.
