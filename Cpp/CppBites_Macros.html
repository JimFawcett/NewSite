C++ Bites: Macros

preprocessor macros and modern alternatives

About C++ Track

1.0 Synopsis

C preprocessor macros (#define) predate the type system and operate as textual substitution before compilation. The preprocessor sees no types, no scopes, no overloads - it performs blind token replacement. Macros are still used for include guards and conditional compilation, but the C++ type system provides safer alternatives for almost every other use.

2.0 Object-like Macros

An object-like macro names a token sequence that the preprocessor substitutes wherever the name appears. 
#define PI 3.14159    // no type, no scope, no debugger visibility

// Modern equivalent: type-safe, scoped, debugger-visible
constexpr double pi = 3.14159265358979;
The #define PI version has no type information, cannot be inspected in a debugger, participates in no namespace, and cannot be overloaded or specialized. Prefer constexpr variables in all cases where the macro is used as a named constant.

3.0 Function-like Macros

Function-like macros accept arguments and substitute them into an expansion. They are prone to subtle bugs because the preprocessor expands arguments by token substitution, not by value. 
// Old: unsafe macro
#define SQUARE(x) ((x) * (x))
int a = 3;
int r = SQUARE(a++);   // undefined behavior: a incremented twice

// New: type-safe constexpr
constexpr double pi = 3.14159265358979;

template<typename T>
constexpr T square(T x) { return x * x; }

static_assert(square(3) == 9);
The macro expansion of SQUARE(a++) produces ((a++) * (a++)) - two increments of a, which is undefined behavior. The template version receives the value of a after a single increment and squares that value safely.

4.0 Include Guards

Include guards prevent a header from being processed more than once per translation unit. They are one of the few remaining legitimate uses of macros in modern C++. 
// Traditional include guard
#ifndef MY_WIDGET_H
#define MY_WIDGET_H

class Widget { /* ... */ };

#endif  // MY_WIDGET_H
The non-standard but universally supported alternative is #pragma once. It is shorter, immune to name collisions, and sufficient for all major compilers (GCC, Clang, MSVC). With C++20 modules, header guards become unnecessary for module interface units. 
#pragma once

class Widget { /* ... */ };

5.0 Conditional Compilation

#ifdef, #if, #elif, and #endif select which code the compiler sees. This is the primary remaining use case where macros have no clean language-level replacement. 
#if defined(_WIN32)
  #include <windows.h>
  using handle_t = HANDLE;
#elif defined(__linux__)
  #include <unistd.h>
  using handle_t = int;
#else
  #error "Unsupported platform"
#endif

#ifdef NDEBUG
  #define ASSERT(expr) ((void)0)
#else
  #define ASSERT(expr) assert(expr)
#endif
C++17 if constexpr replaces many per-type conditional branches inside templates, but platform detection at the file level still requires preprocessor conditionals.

6.0 constexpr and consteval

constexpr (C++11) marks a function or variable as potentially evaluable at compile time. When given constant-expression arguments, the compiler folds the result into the binary as a literal. The function remains callable at runtime with non-constant arguments. 
constexpr int factorial(int n) {
  return n <= 1 ? 1 : n * factorial(n - 1);
}

constexpr int f6 = factorial(6);   // computed at compile time: 720
int           fx = factorial(x);   // computed at runtime for non-const x
consteval (C++20) declares an immediate function that must be evaluated at compile time. Calling it with a non-constant argument is a compile error. 
consteval int compileTimeOnly(int n) { return n * n; }

constexpr int r = compileTimeOnly(5);   // OK: 25
// int y = compileTimeOnly(x);          // error: x not a constant expression

7.0 Stringification and Token Pasting

Two operators are available only inside function-like macro expansions:
  • #arg - stringifies the argument, producing a string literal of its text.
  • a ## b - concatenates two tokens to form a new token.
 
#define STRINGIFY(x) #x
#define CONCAT(a, b) a ## b

std::string s = STRINGIFY(hello world);   // "hello world"
int CONCAT(my, Var) = 42;                 // declares int myVar = 42;

// Practical use: assertion macro that shows failing expression
#define ASSERT_TRUE(expr) \
  do { \
    if (!(expr)) { \
      std::cerr << "Assertion failed: " #expr \
                << " at " __FILE__ ":" << __LINE__ << '\n'; \
      std::abort(); \
    } \
  } while (false)
The do { ... } while (false) wrapper makes the macro behave as a single statement so it works correctly inside if branches without braces.
