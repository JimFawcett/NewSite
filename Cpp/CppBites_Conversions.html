C++ Bites: Conversions

implicit conversions, casts, and explicit constructors

About C++ Track

1.0 Synopsis

C++ supports implicit and explicit type conversions. Implicit conversions happen automatically when the compiler can match types without a cast; explicit conversions require the programmer to write a cast expression. C++ provides four named cast operators — static_cast, dynamic_cast, reinterpret_cast, and const_cast — each with defined semantics and safety characteristics. The C-style cast (Type)expr is best avoided in new code because it silently selects whichever named cast applies.

2.0 Implicit Conversions

Widening conversions (e.g., int to double) are safe and happen automatically. Narrowing conversions (e.g., double to int) truncate silently — a common source of bugs. Brace initialization {} rejects narrowing conversions at compile time, making it the safer initialization form. 
int   i = 3.9;    // compiles: truncated to 3, no warning
// int j{3.9};   // error: narrowing conversion not allowed with {}

double d = 42;    // safe widening: int -> double
float  f = d;     // compiles: precision may be lost

3.0 static_cast

static_cast performs conversions that are verified at compile time. It handles numeric conversions, upcasting and downcasting within a known class hierarchy, and conversions between related pointer/reference types. It will not remove cv-qualifiers or reinterpret memory. Prefer it over C-style casts for all numeric conversions. 
// safe numeric conversion
double d = 3.14;
int    i = static_cast<int>(d);   // 3, truncated

// dynamic_cast: safe polymorphic downcast
Shape* s = new Circle(5.0);
Circle* c = dynamic_cast<Circle*>(s);  // succeeds
if (c) c->draw();

// upcast (always safe, no cast needed but explicit is fine)
Circle  circ(2.0);
Shape&  ref = circ;                // implicit upcast

4.0 dynamic_cast

dynamic_cast performs a runtime-checked downcast within a polymorphic hierarchy. The base class must have at least one virtual function (so RTTI is available). When casting a pointer, it returns nullptr on failure. When casting a reference, it throws std::bad_cast on failure. Use it when the concrete type is genuinely unknown at compile time. 
#include <typeinfo>
#include <stdexcept>

Shape* s = new Rectangle(3.0, 4.0);

// pointer form: returns nullptr on failure
if (Circle* c = dynamic_cast<Circle*>(s)) {
  c->draw();
} else {
  // s is not a Circle
}

// reference form: throws std::bad_cast on failure
try {
  Circle& c = dynamic_cast<Circle&>(*s);
} catch (const std::bad_cast&) {
  // s does not refer to a Circle
}

5.0 reinterpret_cast

reinterpret_cast reinterprets the bit pattern of a value as a different type. It provides no safety checks and easily produces undefined behavior. Legitimate uses are narrow: hardware register access, type-punning via memcpy equivalents, and certain platform ABI boundaries. Avoid it in application code. 
// Reading hardware register at a fixed address (embedded systems)
volatile uint32_t* reg =
  reinterpret_cast<volatile uint32_t*>(0x4000'0000);

// Viewing the byte representation of a float (undefined behavior in general;
// use std::bit_cast (C++20) or memcpy instead)
float f = 1.0f;
uint32_t bits;
std::memcpy(&bits, &f, sizeof bits);  // safe alternative

6.0 const_cast

const_cast adds or removes const (or volatile) from a type. Its only legitimate use is interfacing with legacy C APIs that accept non-const pointers but do not actually modify the data. Writing through a pointer obtained by casting away const from a genuinely const object is undefined behavior. 
// Legacy C API that takes char* but doesn't modify it
void legacy_print(char* s);

const char* msg = "hello";
legacy_print(const_cast<char*>(msg));  // safe: legacy_print won't write

7.0 Explicit Constructors

A single-argument constructor is a converting constructor — the compiler can use it implicitly to convert the argument type to the class type. Marking the constructor explicit disables that implicit conversion and requires the caller to write the construction explicitly. This prevents surprising implicit conversions that compile but produce wrong behavior. 
class Radius {
public:
  /* implicit */ Radius(double r) : r_(r) {}  // allows implicit conversion
  double value() const { return r_; }
private:
  double r_;
};

class SafeRadius {
public:
  explicit SafeRadius(double r) : r_(r) {}
  double value() const { return r_; }
private:
  double r_;
};

void useRadius(Radius r) {}

useRadius(5.0);          // OK: implicit Radius(5.0)
// useRadius(SafeRadius{5.0}); // OK: explicit construction
// useRadius(5.0);       // error if useRadius took SafeRadius
