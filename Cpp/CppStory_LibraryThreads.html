Chapter #13 - std::thread and Concurrency

threads, mutexes, condition variables, and futures

13.0 Prologue

C++11 introduced a portable threading library. std::thread wraps a system thread; std::mutex and friends synchronize access to shared data; std::future and std::async provide higher-level task-based concurrency. All live in <thread>, <mutex>, <condition_variable>, and <future>.

13.1 std::thread

Constructing std::thread starts execution immediately. join() waits for the thread to complete; detach() releases ownership and lets the thread run independently. A joinable thread that goes out of scope without a call to either join() or detach() calls std::terminate - an unconditional program abort. Threads accept any callable: a free function, a lambda, a functor, or a member function pointer paired with an object pointer. Arguments are forwarded to the callable at construction time. C++20 adds std::jthread, which auto-joins on destruction (eliminating the terminate trap) and supports cooperative cancellation through a std::stop_token passed as the first argument to the thread function.
Example: launching two threads with a lambda thread_launch.cpp #include <iostream> #include <thread> #include <string> int main() { auto worker = [](std::string name, int count) { for (int i = 0; i < count; ++i) std::cout << name << ": " << i << "\n"; }; std::thread t1(worker, "alpha", 3); std::thread t2(worker, "beta", 3); t1.join(); t2.join(); std::cout << "both threads finished\n"; } Output (order may vary) alpha: 0 alpha: 1 alpha: 2 beta: 0 beta: 1 beta: 2 both threads finished jthread with stop_token (C++20) #include <iostream> #include <thread> #include <chrono> int main() { std::jthread worker([](std::stop_token st) { int i = 0; while (!st.stop_requested()) { std::cout << "tick " << i++ << "\n"; std::this_thread::sleep_for( std::chrono::milliseconds(50) ); } std::cout << "worker stopping\n"; }); std::this_thread::sleep_for( std::chrono::milliseconds(175) ); worker.request_stop(); // jthread destructor joins automatically } Output tick 0 tick 1 tick 2 worker stopping

13.2 Synchronization

std::mutex protects shared data from concurrent modification. Never lock a mutex manually with lock()/unlock() - use an RAII wrapper so the lock is always released even when an exception unwinds the stack. The two primary wrappers are std::lock_guard (non-movable, simplest) and std::unique_lock (movable, deferrable, required by condition variables). C++17 adds std::scoped_lock, which locks any number of mutexes simultaneously in a deadlock-free order. std::condition_variable lets threads wait for a condition to become true without busy-spinning. wait() atomically releases the associated unique_lock and blocks the calling thread. notify_one() wakes one waiter; notify_all() wakes all of them. Always pass a predicate to wait() to guard against spurious wakeups.
Example: thread-safe queue with mutex and condition_variable SafeQueue.h #include <queue> #include <mutex> #include <condition_variable> #include <optional> template<typename T> class SafeQueue { public: void push(T value) { { std::lock_guard<std::mutex> lk(mtx_); q_.push(std::move(value)); } cv_.notify_one(); } T pop() { std::unique_lock<std::mutex> lk(mtx_); cv_.wait(lk, [this]{ return !q_.empty(); }); T value = std::move(q_.front()); q_.pop(); return value; } bool empty() const { std::lock_guard<std::mutex> lk(mtx_); return q_.empty(); } private: mutable std::mutex mtx_; std::condition_variable cv_; std::queue<T> q_; }; Using SafeQueue #include <iostream> #include <thread> #include "SafeQueue.h" int main() { SafeQueue<int> sq; std::thread producer([&sq]() { for (int i = 0; i < 5; ++i) { sq.push(i); std::cout << "pushed " << i << "\n"; } }); std::thread consumer([&sq]() { for (int i = 0; i < 5; ++i) { int v = sq.pop(); std::cout << "popped " << v << "\n"; } }); producer.join(); consumer.join(); } Output (order may vary) pushed 0 pushed 1 popped 0 popped 1 pushed 2 popped 2 pushed 3 pushed 4 popped 3 popped 4

13.3 Futures and Async

std::async launches a callable as a task and returns a std::future<T> that holds the eventual result. Calling .get() on the future blocks until the result is ready and either returns the value or rethrows any exception the task threw. The launch policy controls scheduling. std::launch::async guarantees a new thread; std::launch::deferred runs the callable lazily in the calling thread when .get() or .wait() is called. The default policy leaves the choice to the implementation. std::promise<T> lets any thread set a future's value manually via set_value() or set_exception(). std::packaged_task wraps a callable so its return value is captured in a future, useful for scheduling work on thread pools or I/O dispatchers.
Example: std::async with exception propagation async_example.cpp #include <iostream> #include <future> #include <stdexcept> #include <numeric> #include <vector> long long sumRange(int lo, int hi) { if (lo > hi) throw std::invalid_argument("lo > hi"); std::vector<int> v(hi - lo + 1); std::iota(v.begin(), v.end(), lo); return std::accumulate( v.begin(), v.end(), 0LL ); } int main() { // launch on a new thread auto f = std::async( std::launch::async, sumRange, 1, 100 ); std::cout << "computing...\n"; try { long long result = f.get(); std::cout << "sum(1..100) = " << result << "\n"; } catch (const std::exception& e) { std::cerr << "error: " << e.what() << "\n"; } // demonstrate exception propagation auto bad = std::async( std::launch::async, sumRange, 10, 1 ); try { bad.get(); } catch (const std::invalid_argument& e) { std::cout << "caught: " << e.what() << "\n"; } } Output computing... sum(1..100) = 5050 caught: lo > hi std::promise example #include <iostream> #include <thread> #include <future> int main() { std::promise<int> prom; std::future<int> fut = prom.get_future(); std::thread t([](std::promise<int> p) { // simulate work std::this_thread::sleep_for( std::chrono::milliseconds(50) ); p.set_value(42); }, std::move(prom)); std::cout << "waiting for promise...\n"; std::cout << "got: " << fut.get() << "\n"; t.join(); } Output waiting for promise... got: 42

13.4 Epilogue

The C++ threading library makes concurrency portable across every major platform. Prefer std::async and std::future for task-level parallelism - the implementation manages thread lifetime for you. Drop to std::thread only when you need precise control over thread affinity, priority, or lifetime. Always protect shared data. Never access a shared variable from multiple threads without synchronization - the C++ memory model gives no guarantees otherwise, and the resulting data races are undefined behavior. Use std::jthread (C++20) in preference to raw std::thread to eliminate the terminate-on-destruction trap.
