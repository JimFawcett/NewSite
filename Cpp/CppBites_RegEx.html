C++ Bites: Regular Expressions

std::regex, matching, searching, and replacing

About C++ Track

1.0 Synopsis

The <regex> header (C++11) provides ECMAScript-syntax regular expressions. The three primary types are std::regex (compiled pattern), std::smatch (match results for std::string), and std::ssub_match (a single capture group within a match). Wide-string variants (std::wregex, std::wsmatch) are also provided.

2.0 Compiling a Pattern

Constructing a std::regex from a string compiles the pattern. The default grammar is ECMAScript. Pass flag constants to select alternatives. 
#include <regex>
#include <string>

// ECMAScript (default)
std::regex digits(R"(\d+)");

// Case-insensitive flag
std::regex word(R"(\bword\b)", std::regex_constants::icase);

// Extended POSIX syntax
std::regex posix("[0-9]+", std::regex_constants::extended);
Raw string literals R"(...)" avoid double-escaping backslashes and make regex patterns significantly more readable.

3.0 regex_match

std::regex_match tests whether the entire string conforms to the pattern. It returns bool. Passing an std::smatch populates capture groups (group 0 is the full match; groups 1..N are the parenthesized sub-expressions). 
std::string date = "2026-05-14";
std::regex  pat(R"((\d{4})-(\d{2})-(\d{2}))");
std::smatch m;

if (std::regex_match(date, m, pat)) {
  std::cout << "year:  " << m[1] << '\n';   // 2026
  std::cout << "month: " << m[2] << '\n';   // 05
  std::cout << "day:   " << m[3] << '\n';   // 14
}
regex_match requires the pattern to match from the first to the last character. Use regex_search when the match may appear anywhere in the string.

4.0 regex_search and regex_replace

#include <regex>
#include <string>
#include <iostream>

int main() {
  std::string text = "user@example.com";
  std::regex email(R"((\w+)@(\w+)\.(\w+))");
  std::smatch m;

  // match_search: find pattern anywhere
  if (std::regex_search(text, m, email)) {
    std::cout << "user: " << m[1] << '\n';  // user
    std::cout << "host: " << m[2] << '\n';  // example
  }

  // regex_replace: substitute match
  std::string masked = std::regex_replace(text, email, "$1@***");
  std::cout << masked << '\n';  // user@***
}
In the replacement string, $0 expands to the full match, $1 to the first capture group, and so on. $$ inserts a literal dollar sign.

5.0 Iterating All Matches

std::sregex_iterator iterates over all non-overlapping matches of a pattern in a string. Dereference it to get an std::smatch. 
std::string text  = "one 1, two 2, three 3";
std::regex  num(R"(\d+)");

auto begin = std::sregex_iterator(text.begin(), text.end(), num);
auto end   = std::sregex_iterator();

for (auto it = begin; it != end; ++it) {
  std::smatch match = *it;
  std::cout << "found: " << match[0] << '\n';
}
// Output:
// found: 1
// found: 2
// found: 3
std::sregex_token_iterator provides a similar interface but yields individual sub-matches or the parts of the string between matches (useful for splitting on a delimiter).

6.0 Regex Replace with Callback

The standard library does not directly support a callable replacement (a function that transforms each match). A common workaround is to use sregex_iterator to collect matches and rebuild the string manually, or to use third-party extensions. 
// Replace each number with its double
std::string text = "a=1, b=2, c=10";
std::regex  num(R"(\d+)");
std::string result;
auto last = text.cbegin();

for (auto it = std::sregex_iterator(text.begin(), text.end(), num);
     it != std::sregex_iterator(); ++it) {
  result.append(last, it->prefix().second);
  int value = std::stoi((*it)[0]);
  result += std::to_string(value * 2);
  last = it->suffix().first;
}
result.append(last, text.cend());
std::cout << result << '\n';   // a=2, b=4, c=20

7.0 Performance Note

Regex compilation is expensive - constructing std::regex from a string parses and compiles the pattern into an internal representation. Never construct a std::regex inside a loop that calls it on many strings. Declare it static or store it as a member or module-level variable. 
// Bad: compiles the regex on every call
bool isDigits(const std::string& s) {
  return std::regex_match(s, std::regex(R"(\d+)"));  // expensive
}

// Good: compile once
bool isDigits(const std::string& s) {
  static const std::regex digits(R"(\d+)");
  return std::regex_match(s, digits);
}
For high-throughput pattern matching, consider PCRE2 or RE2, which offer better performance and more predictable worst-case behavior than the ECMAScript regex engine in most standard library implementations.
