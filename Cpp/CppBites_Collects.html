C++ Bites: Collections

STL containers: sequence, associative, and unordered

About C++ Track

1.0 Synopsis

The Standard Template Library provides a rich set of container classes parameterized on element type. Choosing the right container directly affects performance and code clarity. Containers fall into four categories: sequence containers (ordered by position), associative containers (ordered by key), unordered associative containers (hashed), and container adaptors (interface wrappers over sequence containers).

2.0 Sequence Containers

Sequence containers maintain elements in insertion order. std::vector is a dynamic array with contiguous memory — the right default choice for most use cases. std::array is a fixed-size array on the stack with zero overhead. std::deque supports efficient insertion at both ends. std::list is a doubly-linked list: O(1) insertion anywhere given an iterator, but poor cache locality and no random access. 
#include <vector>
#include <iostream>

int main() {
  std::vector<int> v { 10, 20, 30 };
  v.push_back(40);
  for (auto x : v)
    std::cout << x << ' ';   // 10 20 30 40
}

3.0 Associative Containers

Associative containers store elements sorted by key, backed by a balanced binary tree (typically red-black). std::map<K,V> maps unique keys to values with O(log n) insert, lookup, and erase. std::set<K> stores unique sorted keys with the same complexity. std::multimap and std::multiset allow duplicate keys. 
#include <map>
#include <string>
#include <iostream>

std::map<std::string, int> scores;
scores["Alice"] = 95;
scores["Bob"]   = 87;

if (auto it = scores.find("Alice"); it != scores.end())
  std::cout << it->second;  // 95

4.0 Unordered Containers

Unordered containers use hash tables to achieve O(1) average-case insert, lookup, and erase — faster than their sorted counterparts when ordering is not required and a good hash function is available. std::unordered_map<K,V> and std::unordered_set<K> are the primary types. Worst-case degrades to O(n) on hash collisions. 
#include <unordered_map>
#include <string>
#include <iostream>

std::unordered_map<std::string, int> freq;
freq["apple"]++;
freq["banana"] += 3;

std::cout << freq["apple"]  << '\n';  // 1
std::cout << freq["banana"] << '\n';  // 3

5.0 Container Adaptors

Container adaptors provide restricted interfaces over sequence containers. std::stack<T> wraps a deque (by default) and exposes only push, pop, and top. std::queue<T> enforces FIFO with push/pop/front. std::priority_queue<T> maintains a max-heap, returning the largest element from top(). 
#include <stack>
#include <queue>
#include <priority_queue>

std::stack<int> stk;
stk.push(1); stk.push(2);
stk.top();   // 2

std::priority_queue<int> pq;
pq.push(5); pq.push(1); pq.push(9);
pq.top();   // 9 (max element)

6.0 Choosing a Container

The right container depends on access patterns and requirements:
Container Use when
std::vector default choice; contiguous memory, fast random access and iteration
std::array size known at compile time; zero overhead over a raw array
std::map / std::set ordered unique keys; need lower_bound / upper_bound range queries
std::unordered_map fastest average lookup by key; ordering not needed
std::list frequent insertion/erasure in the middle with stable iterators
std::deque efficient push/pop at both ends
