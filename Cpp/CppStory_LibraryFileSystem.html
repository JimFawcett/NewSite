Chapter #12 - std::filesystem

paths, directory iteration, and file operations

12.0 Prologue

C++17 introduced std::filesystem as a portable replacement for platform-specific path and directory APIs. It was adopted from Boost.Filesystem and standardized under the <filesystem> header. The library covers three broad areas: path manipulation, directory iteration, and file/directory operations.

12.1 Paths

std::filesystem::path represents a file-system path portably. It normalizes separator characters automatically for the host platform, so code written with forward slashes works on Windows, Linux, and macOS. The most useful member functions for path decomposition are:
Member Returns
root_path() root name combined with root directory (e.g., C:\ or /)
parent_path() path without the final component
filename() final path component including extension
stem() filename without extension
extension() extension including leading dot
Paths are concatenated with operator/, which inserts the correct separator: p / "subdir" / "file.txt". A common shorthand is the namespace alias namespace fs = std::filesystem;, which keeps code concise without polluting the global namespace.
Example: Path Decomposition pathDecompose.cpp #include <filesystem> #include <iostream> namespace fs = std::filesystem; int main() { fs::path p = "/home/user/projects/demo.cpp"; std::cout << "root_path : " << p.root_path() << "\n"; std::cout << "parent_path: " << p.parent_path() << "\n"; std::cout << "filename : " << p.filename() << "\n"; std::cout << "stem : " << p.stem() << "\n"; std::cout << "extension : " << p.extension() << "\n"; fs::path joined = fs::path("/home/user") / "projects" / "demo.cpp"; std::cout << "joined : " << joined << "\n"; } Output root_path : "/" parent_path: "/home/user/projects" filename : "demo.cpp" stem : "demo" extension : ".cpp" joined : "/home/user/projects/demo.cpp"

12.2 Directory Iteration

Two iterator types traverse the file system:
Type Behavior
fs::directory_iterator visits only the direct children of the given directory
fs::recursive_directory_iterator descends into all subdirectories depth-first
Each yielded element is a fs::directory_entry, which carries the path and caches status information such as whether the entry is a file or directory. Both iterators support range-based for loops directly. Pass fs::directory_options::skip_permission_denied as a second argument to either iterator constructor to silently skip directories that would raise a permission error, rather than throwing an exception.
Example: Recursive File Listing with Extension Filter listFiles.cpp #include <filesystem> #include <iostream> #include <string> namespace fs = std::filesystem; void listFiles( const fs::path& root, const std::string& ext ) { auto opts = fs::directory_options::skip_permission_denied; for (const auto& entry : fs::recursive_directory_iterator(root, opts)) { if (entry.is_regular_file() && entry.path().extension() == ext) { std::cout << entry.path() << "\n"; } } } int main() { listFiles("/home/user/projects", ".cpp"); } Output (sample) "/home/user/projects/demo/main.cpp" "/home/user/projects/demo/utils.cpp" "/home/user/projects/lib/helpers.cpp"

12.3 File Operations

std::filesystem provides a large set of free functions for querying and manipulating the file system. The table below lists the most commonly used ones.

Table 1. Common std::filesystem Free Functions

Function Description
fs::exists(p) returns true if path exists
fs::is_regular_file(p) returns true if path is a regular file
fs::is_directory(p) returns true if path is a directory
fs::file_size(p) returns size of file in bytes
fs::last_write_time(p) returns last modification time as file_time_type
fs::copy(from, to) copies file or directory (with options for recursion)
fs::copy_file(from, to) copies a single file; overload accepts error_code&
fs::rename(from, to) renames or moves a file or directory
fs::remove(p) removes a single file or empty directory
fs::remove_all(p) removes path and all its contents recursively
fs::create_directory(p) creates a single directory
fs::create_directories(p) creates directory and all missing parents
fs::current_path() returns or sets the current working directory
fs::absolute(p) returns absolute path by prepending current directory if needed
fs::canonical(p) returns absolute path with symlinks and ./.. resolved; path must exist
Most functions have two overloads: one that throws std::filesystem::filesystem_error on failure, and one that accepts a std::error_code& out-parameter and sets it instead of throwing. The error_code overload is preferable in performance-sensitive code or anywhere exceptions are undesirable.
Example: copy_file with error_code copyFile.cpp #include <filesystem> #include <iostream> #include <system_error> namespace fs = std::filesystem; int main() { fs::path src = "input.txt"; fs::path dst = "output.txt"; std::error_code ec; bool ok = fs::copy_file( src, dst, fs::copy_options::overwrite_existing, ec ); if (!ok) { std::cout << "copy failed: " << ec.message() << "\n"; } else { std::cout << "copied " << fs::file_size(dst, ec) << " bytes\n"; } } Output (success case) copied 1024 bytes

12.4 Epilogue

std::filesystem makes file-system work portable and idiomatic. The path type handles separator normalization so the same source code compiles cleanly on Windows, Linux, and macOS. Range-based for over directory_iterator or recursive_directory_iterator is concise and readable. The error_code overloads for every operation let callers opt out of exception overhead where it matters. Together these features replace a large body of platform-specific boilerplate with a uniform, well-specified API.
