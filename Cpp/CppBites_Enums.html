C++ Bites: Enumerations

enum, enum class, and scoped enumerations

About C++ Track

1.0 Synopsis

C++ provides two forms of enumeration. The traditional unscoped enum is C-compatible: enumerators inject their names into the enclosing scope and implicitly convert to int. The modern scoped enum class (C++11) restricts enumerator names to the enum's scope and provides no implicit integer conversion — making it type-safe and less prone to name collisions. Prefer enum class in new code.

2.0 Traditional enum

Enumerators of an unscoped enum are injected directly into the enclosing namespace or scope. They implicitly convert to their underlying integer type (default: int). Two enums in the same scope with identically named enumerators cause a compilation error. 
enum Color { Red, Green, Blue };
// Red, Green, Blue are now in the enclosing scope

Color c = Red;       // OK: Red is visible directly
int   i = Red;       // OK: implicit conversion to int; Red == 0

// Name collision risk:
// enum Fruit { Red, Yellow };  // error: Red already declared

3.0 enum class (Scoped)

Scoped enumerations prefix enumerator names with the enum name, preventing collisions and accidental misuse. There is no implicit conversion to integer — an explicit static_cast is required. The underlying type defaults to int but can be specified explicitly with a trailing : type. 
// unscoped — Red is in enclosing scope
enum Color { Red, Green, Blue };
Color c = Red;    // implicit int: Red == 0

// scoped — must qualify with Direction::
enum class Direction : uint8_t { North, South, East, West };
Direction d = Direction::North;
// int i = d;               // error: no implicit conversion
int i = static_cast<int>(d);  // explicit cast required

4.0 Underlying Type

Both enum and enum class accept a trailing : integral_type to fix the underlying type. This is useful for ABI stability across translation units, forward declarations, controlling storage size in structs, and interfacing with bit-fields or hardware registers. 
enum class Status : uint8_t {
  Ok      = 0,
  Warning = 1,
  Error   = 2,
  Fatal   = 3
};

// Forward declaration (requires explicit underlying type)
enum class Protocol : uint16_t;

// sizeof is now guaranteed
static_assert(sizeof(Status) == 1);

5.0 Switch and Comparison

switch over an enum class requires the fully-qualified enumerator names in each case. The compiler warns (with appropriate flags) about missing cases, which is a valuable safety check. Comparison uses == and != directly; relational operators (<, >) also work because the underlying type supports them. 
Direction d = Direction::East;

switch (d) {
  case Direction::North: /* go north */ break;
  case Direction::South: /* go south */ break;
  case Direction::East:  /* go east  */ break;
  case Direction::West:  /* go west  */ break;
}

if (d == Direction::East)
  std::cout << "heading east\n";
