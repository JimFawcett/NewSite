C++ Bites: Strings

std::string, string_view, and string operations

About C++ Track

1.0 Synopsis

C++ provides two primary string abstractions:
  • std::string - owning, heap-managed, mutable byte string. Lives in <string>.
  • std::string_view - non-owning, read-only reference to an existing string or substring. Lives in <string_view>.
Use std::string when you need to own or modify the text. Use string_view for function parameters that only read a string - it avoids copies and accepts string literals, std::string, and substrings without allocation.

2.0 std::string

std::string is a dynamic sequence of char backed by heap memory. Construction from string literals, concatenation with + or +=, lexicographic comparison with == and <, and substring extraction with substr(pos, len) are all built in. find and rfind return std::string::npos on failure. Small String Optimization (SSO) keeps strings below ~15 characters entirely on the stack, avoiding a heap allocation for short strings. SSO is implementation-defined but standard in all major libraries. 
#include <string>
#include <iostream>

std::string s = "Hello";
s += ", World";
std::cout << s.size() << '\n';    // 12
std::cout << s.substr(7) << '\n'; // World

auto pos = s.find("World");
if (pos != std::string::npos)
  std::cout << "found at " << pos << '\n';  // 7

3.0 string_view

std::string_view is a lightweight (pointer + length) non-owning reference. Constructing one from a string literal or a std::string involves no allocation. It supports the same read-only operations as std::string: size, substr, find, range-for, etc. Dangling view warning: never store a string_view that refers to a temporary std::string - the string may be destroyed while the view is still alive. Use it only as a short-lived parameter or local variable. 
#include <string_view>

// no allocation: just a pointer + length
void print(std::string_view sv) {
  std::cout << sv.size() << ": " << sv << '\n';
}

print("literal");                  // string literal
std::string s = "dynamic";
print(s);                          // std::string
print(std::string_view(s).substr(0, 3)); // "dyn"

4.0 String Literals

C++ supports several literal prefixes and suffixes for strings:
Literal Type Notes
"hello" const char* narrow string literal
"hello"s std::string user-defined literal (C++14), needs using namespace std::string_literals
R"(path\to\file)" const char* raw literal - backslashes and quotes are not escaped
u8"..." const char8_t* UTF-8 encoded (C++20 type)
L"..." const wchar_t* wide string
u"..." const char16_t* UTF-16
U"..." const char32_t* UTF-32 
using namespace std::string_literals;

auto s1 = "hello"s;                    // std::string, no null-terminator issues
auto s2 = R"(C:\Users\jim\file.txt)";  // raw: backslash is literal
auto s3 = R"(He said "hi".)";          // raw: quotes are literal

5.0 Conversions

Convert numeric types to std::string with std::to_string. Parse strings to numbers with std::stoi, std::stol, std::stod, etc. These throw std::invalid_argument if parsing fails and std::out_of_range on overflow. C++20 adds std::format (in <format>) for Python-style string formatting without the overhead of std::ostringstream. 
#include <string>
#include <format>  // C++20

std::string a = std::to_string(42);     // "42"
int         n = std::stoi("123");       // 123
double      d = std::stod("3.14");      // 3.14

// C++20 formatting
std::string msg = std::format("x={}, y={:.2f}", 10, 3.14159);
// "x=10, y=3.14"

Operation Example
length / size s.size(), s.length() (identical)

clear s.clear()

std::string s = "Hello, World!";
s.insert(7, "beautiful ");         // "Hello, beautiful World!"
s.erase(7, 10);                    // "Hello, World!"

if (s.starts_with("Hello"))
  std::cout << "greeting\n";
if (s.ends_with("!"))
  std::cout << "exclamation\n";
