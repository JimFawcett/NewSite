C++ Bites: Threads

std::thread, jthread, and thread-local storage

About C++ Track

1.0 Synopsis

std::thread in <thread> creates OS threads. Every thread must be joined or detached before it is destroyed, or std::terminate is called. C++20 adds std::jthread, which auto-joins on destruction and supports cooperative cancellation via std::stop_token. For shared mutable state, see C++ Bites: Synchronization.

2.0 Creating Threads

Construct a std::thread with any callable (function pointer, lambda, functor) plus its arguments. The callable runs concurrently in the new thread immediately upon construction. Arguments are copied by default; use std::ref to pass by reference. 
#include <thread>
#include <iostream>

void worker(int id) {
  std::cout << "thread " << id << '\n';
}

int main() {
  std::thread t1(worker, 1);
  std::thread t2(worker, 2);
  t1.join();
  t2.join();
}

3.0 join and detach

join() blocks the calling thread until the target thread completes. It must be called exactly once on each joinable thread. detach() releases the thread to run independently as a daemon - it is then no longer joinable. Destroyed std::thread objects that are still joinable call std::terminate. Prefer join. Only detach when the thread's lifetime genuinely extends beyond the launching scope and you have another mechanism to know when it finishes (e.g., a shared atomic flag). 
std::thread t([] {
  // background work
});

if (t.joinable())
  t.join();   // wait for completion

// or:
// t.detach(); // fire and forget - use with caution

4.0 std::jthread (C++20)

std::jthread is a safer wrapper around std::thread. Its destructor calls request_stop() and then join() automatically, eliminating the risk of std::terminate. If the thread function accepts a std::stop_token as its first argument, the token is connected to the jthread's stop source - enabling cooperative cancellation. 
#include <thread>

std::jthread t([](std::stop_token st) {
  while (!st.stop_requested()) {
    // do work
  }
});
// t.request_stop() followed by auto-join on destruction

5.0 Passing Arguments

std::thread copies all arguments by default. Pass a reference with std::ref(x) (from <functional>). Lambdas that capture by reference are equivalent and often clearer, but require the captured variables to outlive the thread. 
#include <thread>
#include <functional>

void fill(std::vector<int>& v, int val) {
  std::fill(v.begin(), v.end(), val);
}

std::vector<int> data(100);

// pass by reference
std::thread t1(fill, std::ref(data), 42);
t1.join();

// equivalent lambda capture
std::thread t2([&data]{ std::fill(data.begin(), data.end(), 7); });
t2.join();

6.0 thread_local

Variables declared with thread_local have thread storage duration: each thread gets its own independent instance, initialized the first time that thread reaches the declaration. This is useful for per-thread caches, random number engines, and C-library errno-style error codes. 
#include <random>

// each thread has its own engine - no synchronization needed
thread_local std::mt19937 rng{std::random_device{}()};

double randDouble() {
  std::uniform_real_distribution<double> dist(0.0, 1.0);
  return dist(rng);
}

7.0 Hardware Concurrency

std::thread::hardware_concurrency() returns the number of logical CPU cores the runtime reports. It may return 0 if the value is not computable. Use it as a starting point for thread pool sizing; tune based on profiling, since the optimal count depends on workload type (CPU-bound vs IO-bound). 
#include <thread>
#include <iostream>

int main() {
  unsigned n = std::thread::hardware_concurrency();
  std::cout << "logical cores: " << (n ? n : 1) << '\n';
}
