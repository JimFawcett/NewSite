Chapter #14 - Windows Sockets (WinSock2)

TCP server and client with the WinSock2 API

14.0 Prologue

Windows Sockets (WinSock2) is the Windows API for TCP/IP and UDP networking. It closely follows the BSD sockets API with a few Windows-specific additions. Before using any WinSock2 facility, include the required headers and link the library: Required headers and lib #define WIN32_LEAN_AND_MEAN #include <winsock2.h> #include <ws2tcpip.h> #pragma comment(lib, "Ws2_32.lib") All WinSock applications must call WSAStartup() before use and WSACleanup() before exit.

14.1 WinSock2 Initialization

WSAStartup(MAKEWORD(2,2), &wsaData) initializes the WinSock2 library for the process. WSACleanup() releases it. The SOCKET type is the handle for a socket; INVALID_SOCKET indicates failure. Use closesocket() to close a socket handle — not the POSIX close().
Example: WSAStartup and WSACleanup Initialization and cleanup #define WIN32_LEAN_AND_MEAN #include <winsock2.h> #include <ws2tcpip.h> #include <iostream> #pragma comment(lib, "Ws2_32.lib") int main() { WSADATA wsaData; int result = WSAStartup(MAKEWORD(2, 2), &wsaData); if (result != 0) { std::cerr << "WSAStartup failed: " << result << "\n"; return 1; } // ... use sockets here ... WSACleanup(); return 0; }

14.2 Server Operations

A TCP server follows this sequence: socket() → bind() → listen() → accept() → send()/recv() → closesocket(). Use getaddrinfo() to resolve a host name and port string into a sockaddr structure suitable for bind() or connect().

Table 1. Key Server Functions

Function Signature Description
socket SOCKET socket(int af, int type, int protocol) Creates a socket. Returns INVALID_SOCKET on failure.
bind int bind(SOCKET s, const sockaddr* name, int namelen) Binds the socket to a local address and port.
listen int listen(SOCKET s, int backlog) Places the socket in a listening state; backlog sets the pending-connection queue length.
accept SOCKET accept(SOCKET s, sockaddr* addr, int* addrlen) Blocks until a client connects, then returns a new socket for that connection.
send int send(SOCKET s, const char* buf, int len, int flags) Sends data on a connected socket. Returns bytes sent or SOCKET_ERROR.
recv int recv(SOCKET s, char* buf, int len, int flags) Receives data from a connected socket. Returns bytes received, 0 on close, or SOCKET_ERROR.
closesocket int closesocket(SOCKET s) Closes a socket handle and releases its resources.

14.3 Client Operations

A TCP client follows this sequence: socket() → connect() → send()/recv() → closesocket(). The example below connects to a server, sends a message, receives a reply, and closes cleanly.
Example: TCP Client TcpClient.cpp #define WIN32_LEAN_AND_MEAN #include <winsock2.h> #include <ws2tcpip.h> #include <iostream> #include <string> #pragma comment(lib, "Ws2_32.lib") int main() { WSADATA wsaData; if (WSAStartup(MAKEWORD(2, 2), &wsaData) != 0) { std::cerr << "WSAStartup failed\n"; return 1; } addrinfo hints{}, *result = nullptr; hints.ai_family = AF_INET; hints.ai_socktype = SOCK_STREAM; hints.ai_protocol = IPPROTO_TCP; if (getaddrinfo("127.0.0.1", "8080", &hints, &result) != 0) { std::cerr << "getaddrinfo failed\n"; WSACleanup(); return 1; } SOCKET sock = socket( result->ai_family, result->ai_socktype, result->ai_protocol ); if (sock == INVALID_SOCKET) { std::cerr << "socket() failed\n"; freeaddrinfo(result); WSACleanup(); return 1; } if (connect(sock, result->ai_addr, (int)result->ai_addrlen) == SOCKET_ERROR) { std::cerr << "connect() failed\n"; closesocket(sock); freeaddrinfo(result); WSACleanup(); return 1; } freeaddrinfo(result); std::string msg = "Hello from client"; send(sock, msg.c_str(), (int)msg.size(), 0); char buf[512]{}; int bytes = recv(sock, buf, sizeof(buf) - 1, 0); if (bytes > 0) { std::cout << "Server replied: " << buf << "\n"; } closesocket(sock); WSACleanup(); return 0; }

14.4 Echo Server Example

The example below is a complete single-threaded TCP echo server. It initializes WinSock2, resolves the bind address with getaddrinfo(), accepts one client at a time, echoes every received buffer back, and cleans up on exit. Port "8080" and INADDR_ANY (via AI_PASSIVE) are used.
Example: Single-Threaded TCP Echo Server EchoServer.cpp #define WIN32_LEAN_AND_MEAN #include <winsock2.h> #include <ws2tcpip.h> #include <iostream> #pragma comment(lib, "Ws2_32.lib") int main() { WSADATA wsaData; if (WSAStartup(MAKEWORD(2, 2), &wsaData) != 0) { std::cerr << "WSAStartup failed\n"; return 1; } addrinfo hints{}, *result = nullptr; hints.ai_family = AF_INET; hints.ai_socktype = SOCK_STREAM; hints.ai_protocol = IPPROTO_TCP; hints.ai_flags = AI_PASSIVE; // bind to INADDR_ANY if (getaddrinfo(nullptr, "8080", &hints, &result) != 0) { std::cerr << "getaddrinfo failed\n"; WSACleanup(); return 1; } SOCKET listenSock = socket( result->ai_family, result->ai_socktype, result->ai_protocol ); if (listenSock == INVALID_SOCKET) { std::cerr << "socket() failed\n"; freeaddrinfo(result); WSACleanup(); return 1; } if (bind(listenSock, result->ai_addr, (int)result->ai_addrlen) == SOCKET_ERROR) { std::cerr << "bind() failed\n"; closesocket(listenSock); freeaddrinfo(result); WSACleanup(); return 1; } freeaddrinfo(result); if (listen(listenSock, SOMAXCONN) == SOCKET_ERROR) { std::cerr << "listen() failed\n"; closesocket(listenSock); WSACleanup(); return 1; } std::cout << "Echo server listening on port 8080\n"; // accept one client at a time (single-threaded) for (;;) { SOCKET clientSock = accept(listenSock, nullptr, nullptr); if (clientSock == INVALID_SOCKET) { std::cerr << "accept() failed\n"; break; } char buf[512]{}; int bytes; while ((bytes = recv(clientSock, buf, sizeof(buf), 0)) > 0) { send(clientSock, buf, bytes, 0); // echo back } closesocket(clientSock); std::cout << "Client disconnected\n"; } closesocket(listenSock); WSACleanup(); return 0; }

14.5 Epilogue

WinSock2 gives direct access to TCP/IP on Windows. The API is low-level; production code wraps it in RAII classes so sockets and WSACleanup() are released deterministically on scope exit. For scalability, production servers use async I/O: WSAAsyncSelect for message-based notification or I/O Completion Ports (IOCP) for high-throughput thread-pool designs. Cross-platform code can target POSIX sockets on Linux and macOS - the call signatures are nearly identical aside from initialization. Replacing the WinSock2 prologue/epilogue and closesocket() with close() covers most of the porting work.
