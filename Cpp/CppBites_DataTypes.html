C++ Bites: Data Types

fundamental types, sizes, and fixed-width integers

About C++ Track

1.0 Synopsis

C++ provides a rich set of fundamental types. Sizes are implementation-defined but bounded by minimum requirements in the standard. On most 64-bit platforms the common widths are well-known, but portable code should never assume exact sizes for int, long, or pointers. Use the fixed-width types from <cstdint> whenever exact width matters for portability, serialization, or hardware interfaces.

2.0 Integer Types

The standard guarantees minimum widths: bool (1 byte), char (1 byte), short (at least 16 bits), int (at least 16 bits, usually 32), long (at least 32 bits), long long (at least 64 bits). Each signed type has an unsigned counterpart. char's signedness is implementation-defined; use signed char or unsigned char explicitly when sign matters.
Type Min width Typical (64-bit)
bool8 bits1 byte
char8 bits1 byte
short16 bits2 bytes
int16 bits4 bytes
long32 bits8 bytes (Linux), 4 bytes (Windows)
long long64 bits8 bytes

3.0 Floating-Point Types

C++ floating-point types follow IEEE 754 on all major platforms. float is 32-bit single precision (~7 significant decimal digits). double is 64-bit double precision (~15 significant digits) — the right default for general computation. long double is 80-bit extended precision on x86 (96 or 128 bytes padded) or 128-bit quad precision on some other architectures. 
float       f = 3.14f;         // 32-bit; note the 'f' suffix
double      d = 3.14159265358; // 64-bit
long double l = 3.14159265358979323846L; // 80 or 128-bit; 'L' suffix

4.0 Fixed-Width Integers

<cstdint> defines exact-width types (int8_t through int64_t and their unsigned variants), minimum-width types (int_least8_t, etc.), and fastest-minimum types (int_fast8_t, etc.). Use exact-width types for binary protocols, file formats, and hardware registers. 
#include <cstdint>
#include <limits>
#include <iostream>

int32_t  counter = 0;
uint64_t mask    = 0xFF'FF'FF'FF'FF'FF'FF'FFull;

std::cout << sizeof(int32_t)  << '\n';   // 4
std::cout << sizeof(uint64_t) << '\n';   // 8
std::cout << std::numeric_limits<int32_t>::max() << '\n'; // 2147483647

5.0 auto and decltype

auto deduces the type of a variable from its initializer, following the same rules as template argument deduction. It drops top-level cv-qualifiers and references from the initializer's type — use auto& or const auto& to keep them. decltype(expr) yields the exact type of an expression without evaluating it, preserving references and cv-qualifiers. Neither replaces the need to understand the underlying type; they reduce repetition where the type is already clear from context. 
#include <vector>

std::vector<int> v{1, 2, 3};

auto        i = v[0];           // int (copy)
auto&       r = v[0];           // int& (reference)
const auto& cr = v[0];          // const int&

decltype(v[0]) x = v[0];        // int& — preserves reference from operator[]
decltype(42)   y = 0;           // int

6.0 Type Sizes and Limits

sizeof(T) returns the size in bytes at compile time. alignof(T) returns the required alignment in bytes. std::numeric_limits<T> (from <limits>) provides min, max, digits, infinity, and other properties for any numeric type. 
#include <limits>
#include <iostream>

std::cout << sizeof(double)   << '\n';   // 8
std::cout << alignof(double)  << '\n';   // 8

std::cout << std::numeric_limits<double>::max()     << '\n';
std::cout << std::numeric_limits<double>::epsilon() << '\n';
std::cout << std::numeric_limits<int>::min()        << '\n';  // -2147483648
