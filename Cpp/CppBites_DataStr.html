C++ Bites: Data Structures

struct layouts, aggregate initialization, and member alignment

About C++ Track

1.0 Synopsis

C++ structs and classes model data structures directly in memory. Understanding memory layout — padding, alignment, and member ordering — helps write efficient, portable code and avoid subtle bugs when interfacing with hardware, file formats, or network protocols. The standard also provides std::pair and std::tuple as generic heterogeneous data holders.

2.0 Struct vs Class

The only technical difference between struct and class in C++ is default member access: struct members default to public, class members default to private. Convention: use struct for plain data aggregates with no invariants, and class for types that encapsulate state and enforce invariants via constructors and accessors. 
// Plain data: struct is idiomatic
struct Point { double x, y, z; };

// Invariant-enforcing type: class is idiomatic
class Circle {
public:
  explicit Circle(double r) : radius_(r) { /* assert r > 0 */ }
  double area() const { return 3.14159 * radius_ * radius_; }
private:
  double radius_;
};

3.0 Aggregate Initialization

An aggregate is a class/struct with no user-provided constructors, no private or protected non-static data members, no base classes (pre-C++17), and no virtual functions. Aggregates support brace initialization, which initializes members in declaration order. C++20 adds designated initializers, allowing members to be initialized by name in any subset. 
struct Point { double x, y, z; };

Point p1 = {1.0, 2.0, 3.0};       // aggregate init
Point p2{.x=1.0, .y=2.0, .z=3.0}; // C++20 designated

auto [x, y, z] = p2;              // structured binding

4.0 Padding and Alignment

The compiler inserts padding bytes between members to align each member on its natural alignment boundary (typically equal to its size). The struct's total size is then padded to a multiple of its largest member's alignment. Ordering members from largest to smallest minimizes padding and reduces struct size — important in large arrays or cache-sensitive code. 
struct Bad  { char a; int b; char c; };  // sizeof: 12
struct Good { int b; char a; char c; };  // sizeof: 8

static_assert(sizeof(Bad)  == 12);
static_assert(sizeof(Good) ==  8);

// alignas forces a minimum alignment
struct alignas(64) CacheLine {
  int data[16];  // occupies one 64-byte cache line
};

5.0 std::pair and std::tuple

std::pair<T1, T2> bundles two heterogeneous values; members are accessed as .first and .second. std::tuple<Ts...> generalizes to N values; use std::get<N>(t) or structured bindings to unpack. Both support comparison operators when the element types do. 
#include <utility>
#include <tuple>
#include <string>
#include <iostream>

auto pair = std::make_pair(42, std::string("hello"));
std::cout << pair.first << ' ' << pair.second << '\n';

auto [a, b] = pair;  // structured binding

std::tuple<int, double, std::string> t{1, 3.14, "world"};
std::cout << std::get<0>(t) << '\n';   // 1
auto [i, d, s] = t;                     // unpack all three
