C++ Bites: Async and Await

std::async, std::future, and std::promise

About C++ Track

1.0 Synopsis

C++ provides asynchronous execution through std::async, std::future, and std::promise in the <future> header. Unlike the coroutine-based async/await found in languages like C# or JavaScript, C++ futures represent a value that will be available at some later point. The caller launches work, continues with other computation, then blocks on future.get() only when the result is actually needed.

2.0 std::async

std::async launches a callable asynchronously and returns a std::future<T> that holds the eventual result. The launch policy controls scheduling: std::launch::async runs the callable on a new thread immediately; std::launch::deferred delays execution until get() or wait() is called on the future (lazy evaluation on the calling thread). Omitting the policy lets the implementation choose. 
#include <future>
#include <iostream>

int compute(int x) { return x * x; }

int main() {
  std::future<int> f = std::async(std::launch::async, compute, 7);
  // do other work while compute runs
  int result = f.get();   // blocks until ready
  std::cout << result << '\n';  // 49
}

3.0 std::promise and std::future

A std::promise<T> and its associated std::future<T> form a one-shot channel between threads. The producer calls promise.set_value() to fulfill the promise; the consumer calls future.get() to retrieve the value, blocking if necessary. A promise can also propagate an exception via promise.set_exception(). 
#include <future>
#include <thread>
#include <iostream>

int main() {
  std::promise<int> prom;
  std::future<int>  fut = prom.get_future();

  std::thread producer([&prom]() {
    // simulate work
    int answer = 42;
    prom.set_value(answer);
  });

  std::cout << fut.get() << '\n';  // blocks until producer sets value
  producer.join();
}

4.0 std::packaged_task

std::packaged_task<F> wraps a callable and connects it to a future. Unlike std::async, it does not launch automatically — you invoke it explicitly, often by passing it to a thread. This gives more control over when and where execution occurs while still providing a future for the result. 
#include <future>
#include <thread>

int compute(int x) { return x * x; }

int main() {
  std::packaged_task<int(int)> task(compute);
  std::future<int> fut = task.get_future();

  std::thread t(std::move(task), 9);  // runs compute(9)
  t.join();

  return fut.get();  // 81
}

5.0 Exception Propagation

If the callable passed to std::async (or fulfilled via set_exception) throws, the exception is captured and stored in the shared state. Calling future.get() re-throws it at the call site. This means async exceptions surface naturally in the consuming thread without any manual exception-transport code. 
#include <future>
#include <stdexcept>
#include <iostream>

int risky(int x) {
  if (x < 0) throw std::invalid_argument("negative input");
  return x * x;
}

int main() {
  auto f = std::async(std::launch::async, risky, -1);
  try {
    f.get();  // re-throws std::invalid_argument
  } catch (const std::exception& e) {
    std::cerr << "Caught: " << e.what() << '\n';
  }
}
