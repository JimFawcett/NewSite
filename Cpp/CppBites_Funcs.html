C++ Bites: Functions

free functions, overloading, lambdas, and function objects

About C++ Track

1.0 Synopsis

Functions are the first-class building blocks of C++ programs. C++ supports free functions, member functions, overloaded functions, function templates, lambdas, and std::function. Each form serves a different purpose: free functions for global operations, overloading for uniform interfaces across types, lambdas for local callables, templates for generic algorithms, and std::function for type-erased storage of any callable.

2.0 Free Functions

A free function is declared at namespace scope. Its signature consists of a return type, a name, and a parameter list. Parameters may have default arguments; those must appear at the end of the parameter list and be consistent across all declarations in the same translation unit. 
// Declaration (in header)
int add(int a, int b = 0);

// Definition
int add(int a, int b) {
  return a + b;
}

// Call sites
int x = add(3, 4);   // 7
int y = add(5);      // 5  (uses default b = 0)
Parameters are passed by value by default. Pass by const reference to avoid copies of large objects: void print(const std::string& s). Pass by pointer only when the argument is optional (nullable).

3.0 Methods

A method (member function) is declared inside a class or struct and has implicit access to the object's data via the this pointer. Qualifying a method const promises it will not modify the object; only const methods may be called on const instances or through const references. 
struct Counter {
  int count = 0;

  void increment()       { ++count; }
  int  value()    const  { return count; }
};

Counter c;
c.increment();
c.increment();
std::cout << c.value();  // 2

const Counter& ref = c;
// ref.increment();       // error: calls non-const method on const ref
std::cout << ref.value(); // ok
Methods defined inside the class body are implicitly inline. Definitions placed outside the class use the qualified name: int Counter::value() const { ... }.

4.0 Overloading

Multiple functions may share the same name as long as their parameter lists differ in type, count, or both. The compiler selects the best match at call time using overload resolution - a compile-time process that considers implicit conversions and template argument deduction. Return type alone does not distinguish overloads. 
void display(int n)         { std::cout << "int: " << n << '\n'; }
void display(double d)      { std::cout << "double: " << d << '\n'; }
void display(const char* s) { std::cout << "string: " << s << '\n'; }

display(42);      // calls display(int)
display(3.14);    // calls display(double)
display("hello"); // calls display(const char*)
Overload resolution follows a ranked set of rules: exact match beats promotion beats standard conversion beats user-defined conversion. Ambiguous calls that cannot be resolved uniquely produce a compile error.

5.0 Lambdas

A lambda is an unnamed function object defined inline. The syntax is [capture](params) -> RetType { body }. The return type is inferred when omitted. The capture clause controls which names from the enclosing scope the lambda body can access. Capture modes:
  • [=] - capture all referenced locals by value (copy at lambda creation).
  • [&] - capture all referenced locals by reference (dangling risk if lambda outlives scope).
  • [x, &y] - capture x by value and y by reference.
  • [] - capture nothing; the lambda can only use its parameters and globals.
 
auto square = [](int x) { return x * x; };
auto add    = [](auto a, auto b) { return a + b; };  // generic lambda

int y = 10;
auto addY = [y](int x) { return x + y; };  // capture by value

std::function<int(int)> fn = square;
std::cout << fn(5);  // 25
Since C++14, lambda parameters may be auto, producing a generic lambda whose call operator is itself a template. Since C++20, explicit template parameter lists are allowed: []<typename T>(T x) { ... }.

6.0 Function Templates

A function template is a recipe for generating a family of functions. The compiler instantiates a concrete function for each unique set of template arguments it encounters. Type deduction eliminates the need to specify arguments explicitly in most cases. 
template<typename T>
T max(T a, T b) {
  return (a > b) ? a : b;
}

int    i = max(3, 7);          // T = int, deduced
double d = max(1.2, 3.4);      // T = double, deduced
auto   s = max<std::string>("alpha", "beta");  // T explicit

// Explicit (full) specialization for const char*
template<>
const char* max<const char*>(const char* a, const char* b) {
  return std::strcmp(a, b) > 0 ? a : b;
}
C++20 Concepts constrain template parameters: template<std::totally_ordered T> T max(T a, T b) rejects types that lack operator> at instantiation time with a clear error message.

7.0 std::function and std::invoke

std::function<Sig> is a type-erased wrapper that stores any callable (free function pointer, lambda, functor, member function pointer bound via std::bind) with a compatible signature. It is copy-constructible and assignable, making it suitable for callbacks stored in containers or class members. 
#include <functional>

// Store a free function
std::function<int(int, int)> op = add;

// Replace with a lambda
op = [](int a, int b) { return a * b; };
std::cout << op(3, 4);  // 12

// std::invoke: uniform call syntax for any callable
struct Obj { int value(int x) { return x * 2; } };
Obj obj;
int r = std::invoke(&Obj::value, obj, 5);  // 10
std::function incurs a heap allocation for large callables and a virtual dispatch on each call. Prefer template parameters or auto for performance-critical callbacks. C++23 std::move_only_function removes the CopyConstructible requirement.
