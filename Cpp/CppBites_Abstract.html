C++ Bites: Abstract Classes

pure virtual functions, abstract base classes, and polymorphic hierarchies

About C++ Track

1.0 Synopsis

An abstract class is a class that has at least one pure virtual function declared with the syntax virtual ReturnType f() = 0;. An abstract class cannot be instantiated directly — it serves as an interface contract that derived classes must fulfill. Any class that inherits from an abstract base but does not override all pure virtual functions is itself abstract. Abstract classes enable polymorphism: code written against the base class interface works uniformly with any conforming derived type.

2.0 Pure Virtual Functions

A pure virtual function is declared by appending = 0 to the virtual function declaration. Derived classes must override every pure virtual function to become concrete. The override keyword on the derived class is not required but is strongly recommended — it lets the compiler verify that the function actually overrides a base class virtual. 
class Shape {
public:
  virtual double area() const = 0;   // pure virtual
  virtual void draw() const = 0;
  virtual ~Shape() = default;        // must be virtual
};

class Circle : public Shape {
public:
  explicit Circle(double r) : radius_(r) {}
  double area() const override { return 3.14159 * radius_ * radius_; }
  void draw() const override { /* render circle */ }
private:
  double radius_;
};

3.0 Abstract Base Classes

The Shape class above is an abstract base class (ABC). It declares the interface that all shapes must implement. Circle and Rectangle are concrete types that provide implementations for every pure virtual function. 
class Rectangle : public Shape {
public:
  Rectangle(double w, double h) : width_(w), height_(h) {}
  double area() const override { return width_ * height_; }
  void draw() const override { /* render rectangle */ }
private:
  double width_, height_;
};

// Shape s;  // error: cannot instantiate abstract class
Circle    c(5.0);
Rectangle r(3.0, 4.0);

4.0 Virtual Destructors

Any class that serves as a polymorphic base must declare its destructor virtual. Without it, deleting a derived object through a base pointer invokes undefined behavior — only the base destructor runs and the derived class resources leak. 
// Correct: virtual destructor ensures full cleanup
class Shape {
public:
  virtual ~Shape() = default;  // non-pure virtual destructor
  virtual double area() const = 0;
  virtual void draw() const = 0;
};

// Safe deletion through base pointer
Shape* s = new Circle(5.0);
delete s;  // calls ~Circle(), then ~Shape()

5.0 Polymorphism

The canonical C++ pattern for polymorphism stores derived objects through base pointers or references. std::unique_ptr<Shape> manages ownership safely and calls the correct destructor automatically. Virtual dispatch selects the right override at runtime. 
#include <vector>
#include <memory>
#include <iostream>

int main() {
  std::vector<std::unique_ptr<Shape>> shapes;
  shapes.push_back(std::make_unique<Circle>(5.0));
  shapes.push_back(std::make_unique<Rectangle>(3.0, 4.0));
  shapes.push_back(std::make_unique<Circle>(2.0));

  for (const auto& s : shapes) {
    std::cout << s->area() << '\n';  // virtual dispatch
    s->draw();
  }
  // unique_ptrs call virtual destructor on scope exit
}
