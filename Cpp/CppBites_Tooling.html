C++ Bites: Tooling

compilers, CMake, VS Code, vcpkg, debuggers, and static analysis

About C++ Track

1.0 Synopsis

The C++ ecosystem includes compilers, build system generators, package managers, debuggers, static analyzers, and runtime sanitizers. Mastering these tools is as important as mastering the language itself. The typical modern stack: compiler (MSVC, GCC, or Clang) + CMake + vcpkg + clang-tidy + sanitizers.

2.0 Compilers

Compiler Executable Primary Platform Notes
MSVC cl.exe Windows Visual Studio; /std:c++20, /W4, /O2
GCC g++ Linux / macOS / MinGW -std=c++20 -Wall -Wextra -O2
Clang clang++ cross-platform best diagnostics; required for clang-tidy and LLDB
All three compilers fully support C++20 and have partial C++23 support as of 2024. Use -std=c++20 (GCC/Clang) or /std:c++20 (MSVC). Always enable warnings: -Wall -Wextra on GCC/Clang or /W4 on MSVC. Add -Werror / /WX in CI to treat warnings as errors.

3.0 CMake

CMake is the de facto build-system generator for C++. It produces Makefiles, Ninja build files, or Visual Studio solutions from a CMakeLists.txt description. The typical workflow: cmake -B build -DCMAKE_BUILD_TYPE=Release then cmake --build build. 
cmake_minimum_required(VERSION 3.20)
project(MyProject)

set(CMAKE_CXX_STANDARD 20)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

add_executable(myapp src/main.cpp)
target_compile_options(myapp PRIVATE -Wall -Wextra)
Use target_include_directories, target_link_libraries, and target_compile_definitions to express per-target properties. Avoid setting directory-wide properties with include_directories or add_definitions - they leak into subdirectories.

4.0 Visual Studio Code

VS Code is a free, cross-platform editor that competes with full IDEs for C++ development once the right extensions are installed. The essential extensions are:
  • ms-vscode.cpptools - IntelliSense, code navigation, and the integrated debugger
  • ms-vscode.cmake-tools - CMake configure, build, and test from the status bar
  • ms-vscode.cpptools-extension-pack - bundles both of the above plus themes and remote dev
  • llvm-vs-code-extensions.vscode-clangd - faster, more accurate IntelliSense via clangd (disable cpptools IntelliSense if used)
 A typical C++ workspace view after opening a CMake project:
VS Code - C++ workspace
 The CMake Tools extension writes a compile_commands.json into the build directory. Point clangd at it via .vscode/settings.json to get accurate cross-file navigation and diagnostics without configuring include paths manually. 
// .vscode/settings.json
{
  "clangd.arguments": ["--compile-commands-dir=${workspaceFolder}/build"],
  "cmake.buildDirectory": "${workspaceFolder}/build",
  "cmake.configureOnOpen": true
}

5.0 vcpkg

vcpkg is Microsoft's C++ package manager. It installs pre-built or source-compiled libraries and integrates with CMake via a toolchain file. In manifest mode (recommended), dependencies are declared in a vcpkg.json file and installed automatically during the CMake configure step. 
# install vcpkg (one time)
git clone https://github.com/microsoft/vcpkg
.\vcpkg\bootstrap-vcpkg.bat

# install a library
.\vcpkg\vcpkg install fmt

# integrate with CMake
cmake -B build -DCMAKE_TOOLCHAIN_FILE=path/to/vcpkg/scripts/buildsystems/vcpkg.cmake
In CMakeLists.txt, find and link the installed library as normal: 
find_package(fmt CONFIG REQUIRED)
target_link_libraries(myapp PRIVATE fmt::fmt)

6.0 Debuggers

Debugger Platform Usage
Visual Studio debugger Windows integrated; press F5; excellent for MSVC and CMake projects
GDB Linux gdb ./myapp; run, break, next, print
LLDB macOS / Clang similar command set to GDB; integrates with Xcode
VS Code C++ extension cross-platform drives GDB or LLDB via launch.json; works with CMake Tools

7.0 Static Analysis

Static analysis tools examine source code without running it, finding potential bugs, style violations, and security issues at zero runtime cost. clang-tidy - enforces a configurable set of C++ Core Guidelines and lint checks. Configure via .clang-tidy. Integrate with CMake: 
set(CMAKE_CXX_CLANG_TIDY clang-tidy;--warnings-as-errors=*)
cppcheck - finds uninitialized variables, out-of-bounds reads, memory leaks, and other bugs statically. Runs separately from the build. MSVC /analyze - built-in static analysis in the MSVC compiler; enabled with /analyze in project properties.

8.0 Sanitizers

Sanitizers instrument compiled code to detect runtime errors that are otherwise silent or non-deterministic. They impose a runtime overhead (typically 2-5x) but are invaluable in debug and CI builds.
Sanitizer Flag (GCC/Clang) MSVC Flag Detects
AddressSanitizer (ASAN) -fsanitize=address /fsanitize=address heap/stack overflows, use-after-free, double-free
UndefinedBehaviorSanitizer (UBSAN) -fsanitize=undefined - signed overflow, null deref, invalid shifts
ThreadSanitizer (TSAN) -fsanitize=thread - data races
MemorySanitizer (MSAN) -fsanitize=memory - uninitialized reads (Clang only) 
g++ -std=c++20 -g -fsanitize=address,undefined -o myapp src/main.cpp
./myapp   # errors printed at runtime with file/line info
