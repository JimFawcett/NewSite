Basics: Plans

platforms, languages, code

About Track Summary
Plans for this "Basics Track" focus on additional platform details, language history and relationships, and code repositories, demos, tools, and testing.

Platforms

  • Expand discussion of platform types, history, and relationships.
  • Discuss exemplar platforms in move detail, focusing on the Posix specification and Widnows API.
  • Platform APIs
    • API vs Library
    • POSIX specification
    • macOS - Certified POSIX complient
    • Linux - Fully POSIX complient
    • Windows 11 - non-POSIX complient
  • Explore processes and threads for Linux and Windows 11.
    • Linux threads are processes that inherit their parent's address space. pthreads is a library that abstracts away process details. Linux schedules processes.
    • Windows threads are the unit of scheduling. Processes are containers for threads and hold their context.
  • Expand discussion of synch and asynch I/O, especially completion ports.
  • Expand discussion of system resources:
    • Files
    • Sockets and Pipes
    • Mutex, Semiphore
    • Module, Process, Thread, Timer
    • Desktop, Console, Heap, Job
  • Security

Languages

    • Native vs managed - both eventually native, but very different codes
  • Examples

    • Declarative HTML, CSS, SQL, ...
    • Functional Haskell, Cloture, Rust iterators, ...
  • Asynchrony

    • Blocking queues
    • Thread pools
    • Coroutines
    • async functions
    • Tasks
  • Patterns
    • Design Object Factory, Observer, Strategy, MixIn, Reactor/Proactor, ...
    • Implementation SOLID, DRY, ...
