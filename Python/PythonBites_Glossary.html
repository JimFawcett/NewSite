PythonBites: Glossary

Python keywords and terms

About Python Track

A

Argument
A value passed to a function. Can be positional or keyword-based.
Async/Await
Keywords used for defining asynchronous functions and handling coroutines.
Attribute
A value or method attached to an object, accessed using dot notation (obj.attr).
Abstract Base Class (ABC)
A class that cannot be instantiated and defines an interface.

B

Boolean
A type with two values: True and False.
Built-in Functions
Functions always available in Python (e.g., len(), range(), print()).
Bytecode
Intermediate representation of Python source code executed by the interpreter.

C

Class
A user-defined blueprint for creating objects.
Comprehensions
Concise syntax for generating lists, sets, or dicts ([x for x in iterable]).
Context Manager
Object that defines __enter__ and __exit__, used with with statements.
Coroutine
A function defined with async def that can be paused and resumed.

D

Decorator
A function that modifies another function or class behavior (@decorator syntax).
Dictionary
A key-value mapping data structure ({key: value}).
Duck Typing
Pythonic behavior where an objectâ€™s type is less important than its behavior.

E

Exception
An error detected during execution that disrupts the normal flow of a program.
Ellipsis (...)
Special object often used as a placeholder or in slicing multi-dimensional arrays.

F

Function
A block of reusable code, defined using def or lambda.
First-Class Object
All Python objects can be assigned to variables, passed as arguments, or returned.

G

Generator
A function that yields values lazily using yield.
Global Variable
A variable declared outside any function, accessible throughout the module.
GIL (Global Interpreter Lock)
A mutex in CPython preventing multiple native threads from executing Python bytecode simultaneously.

H

Hashable
An object with a hash value that does not change during its lifetime (used as dict keys).
Heap
The area of memory where Python stores dynamically allocated objects.

I

Import
The mechanism to include code from other modules using import, from, and as.
Immutable
An object whose state cannot be modified after creation (e.g., tuple, str).
Iterable
An object capable of returning its members one at a time (e.g., list, generator).

J

JSON (JavaScript Object Notation)
Common data interchange format, supported via the json module.

K

Keyword Argument
Argument passed by explicitly naming the parameter (func(x=1)).
Keyword
Reserved words that cannot be used as identifiers (e.g., if, class, def, lambda).

L

Lambda
Anonymous function defined with the lambda keyword.
List
Mutable sequence type ([1, 2, 3]).
List Comprehension
Concise way to create a list from an iterable.

M

Module
A file containing Python definitions and code, which can be imported.
Mutable
Object whose internal state can be changed (e.g., list, dict, set).

N

Namespace
A container that holds a mapping between names and objects.
None
A special constant representing the absence of a value or null.

O

Object
Any data in Python is an object, including functions and types.
Operator Overloading
Defining special methods (e.g., __add__) to enable custom behavior with operators.

P

Package
A directory with an __init__.py file, containing one or more modules.
PEP (Python Enhancement Proposal)
Design document for introducing new features or standards.
Pythonic
Code or style that follows idiomatic Python principles.

Q

Queue
Data structure that follows FIFO order; implemented in queue or collections.deque.

R

REPL
Read-Eval-Print Loop; an interactive shell for executing Python code.
Range
Built-in function that generates a sequence of numbers (range(start, stop, step)).

S

Set
An unordered collection of unique elements ({1, 2, 3}).
Slice
A subset of a sequence (seq[start:stop:step]).
Static Method
Method bound to a class rather than its object, declared with @staticmethod.

T

Tuple
Immutable sequence of values ((1, 2, 3)).
Type Hint
Syntax for specifying the expected type of variables, parameters, and return values.
Traceback
Error message showing the call stack at the point of exception.

U

Unpacking
Assignment of elements in a sequence to variables (a, b = (1, 2)).
Underscores
Special naming convention; e.g., _private, __magic__.

V

Variable
A symbolic name bound to an object.
Virtual Environment
A self-contained directory for managing project-specific dependencies (venv, virtualenv).

W

While Loop
Repeats a block of code while a condition remains true.
With Statement
Simplifies exception handling by encapsulating common try/finally patterns (e.g., file handling).

Y

Yield
Pauses a generator function and returns a value; resumes on next iteration.
