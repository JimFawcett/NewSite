1. What is SWDev?

The Software Development (SWDev) track is concerned with the processes, tools, and artifacts involved in taking a software idea from concept to deployed product. It is intentionally practical - focused on things that matter in professional work. Three defining activities:
  • Design - developing the concept, architecture, and specification for a software system, and documenting those decisions so the development team can implement and maintain them.
  • Implementation - writing, organizing, and testing code in modules and libraries, guided by the design and verified by test suites.
  • Deployment - moving finished software to where it will be used, using version control, automation scripts, containers, and cloud or local platforms.
The track uses a concrete example project - TextFinder - throughout its pages to keep discussion grounded. TextFinder finds regex matches in a directory tree and is implemented in Rust, C++, C#, and Python, making it a good vehicle for comparing design and implementation strategies across languages.

3. Key Concepts

The track covers the areas below, roughly in the order they arise during development.

Software Design

Good design starts with understanding what the software must do, who will use it, and what constraints (performance, cost, schedule) apply.
  • Concept development - goals, users, scope, and foreseeable issues
  • Architecture - decomposition into packages and components with defined interfaces
  • Specification - precise statement of what each component must do
  • Documentation - design documents, interface tables, and inline comments that explain non-obvious decisions
Pages: Design Bites Introduction, Design Overview.

Structural Design Patterns

Structure Bites explore five ways to organize a software system, using TextFinder as a running example to make the tradeoffs concrete.
  • Basic structure - linear package dependencies with clear data flow
  • Data flow structure - components connected by well-typed event channels
  • Factored structure - separating policy from mechanism through abstraction
  • Type erasure - hiding concrete types behind trait objects or abstract base classes
  • Plugin structure - loading behavior at runtime via dynamic dispatch or shared libraries
Pages: Basic, Data Flow, Factored, Type Erasure, Plugin.

Development Process and Deployment

Deploy Bites cover the practical tools and workflows used to build, manage, and ship software in a professional setting.
  • Process models - waterfall gives predictability; iterative and agile accommodate change; scrum provides a lightweight team rhythm
  • Configuration management - git branches, commits, merges, and pull requests for tracking every change to the codebase
  • Scripting - Bash and PowerShell for automating builds, tests, and deployments
  • Containers - Docker and similar tools package all dependencies so software runs identically in development and production
  • Platforms - GitHub for remote code management, CI pipelines for automated testing on every push
Pages: Process, Git.

Example Projects

Concrete projects anchor the abstract concepts throughout the track.
  • TextFinder - find regex matches in a directory tree; command-line arguments specify the root path, regex pattern, and file extensions. Implemented in Rust, C++, C#, and Python for direct comparison.
  • CodeAnalyzer - analyze source files in a directory tree; find user-defined types, display public interfaces, and report size and complexity by function. Designed using Rule and Action interfaces or traits.
Pages: SWDev Projects, SWDev Repositories.

4. Getting Started

Recommended first steps for someone new to the SWDev track:
  1. Read the Design Bites Introduction. It lays out the full scope of the design track and introduces the TextFinder project that ties the pages together.
  2. Work through the Structure Bites in order - starting with Basic Structure and progressing to Data Flow, Factored, Type Erasure, and Plugin. Each builds on the previous and uses the same example project.
  3. Read the Deploy Bites Introduction for an overview of deployment tools and processes. Then read Process and Git as the most immediately practical pages.
  4. Browse the Projects page to see full project specifications and links to implementations in multiple languages. Comparing the same design across Rust, C++, C#, and Python builds intuition about language-level tradeoffs.
  5. Use the Glossary and References as reference material throughout.
